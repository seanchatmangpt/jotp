# Event-Driven Architecture

Learn how to build scalable, loosely coupled systems using JOTP's typed event bus and reactive patterns.

## What You'll Build

A real-time analytics system that:
- Publishes domain events
- Subscribes to event types
- Handles failures independently
- Scales horizontally

## Core Concepts

- **EventManager<E>**: Typed pub-sub event bus
- **Event Handlers**: Isolated processes for each subscriber
- **Loose Coupling**: Publishers don't know about subscribers
- **Fault Isolation**: Handler crashes don't affect the bus

## Event Flow

```
Publisher → EventManager → [Handler-1, Handler-2, Handler-3]
                              ↓           ↓           ↓
                          Analytics   Logging    Alerting
```

## Implementation Steps

### 1. Define Event Types
```java
public sealed interface DomainEvent
    permits OrderCreated, PaymentCompleted, InventoryUpdated {
    record OrderCreated(String orderId, BigDecimal amount) implements DomainEvent {}
    // ...
}
```

### 2. Create Event Bus
- Instantiate `EventManager<DomainEvent>`
- Subscribe handlers with filtering
- Publish events from business logic

### 3. Build Handlers
- Each handler is a separate `Proc`
- Process events asynchronously
- Emit new events as needed

## Advanced Patterns

- **CQRS**: Separate read/write models via events
- **Event Sourcing**: Rebuild state from event log
- **Saga Orchestration**: Coordinate distributed transactions
