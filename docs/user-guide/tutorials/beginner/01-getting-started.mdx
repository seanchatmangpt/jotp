# Getting Started with JOTP

<Badge variant="secondary">Tutorial 1 of 6</Badge>

Welcome to JOTP! This tutorial will guide you through setting up your development environment and creating your first fault-tolerant process. By the end, you'll have a running JOTP application that demonstrates the power of supervised processes.

## What You'll Learn

In this tutorial, you'll:

- Set up a Java 26 development environment with preview features
- Add JOTP to your project
- Create your first supervised process
- See the "let it crash" philosophy in action
- Understand basic process communication

## Prerequisites

Before we begin, make sure you have:

- **Java 26** installed with preview features enabled
- **Maven 4** or higher (or Maven Wrapper)
- A code editor (IntelliJ IDEA, VS Code, or similar)
- Basic familiarity with Java

## Step 4: Your First JOTP Process

Let's create a simple counter process that demonstrates JOTP's core concepts.

Create the file `src/main/java/com/example/quickstart/QuickStart.java`:

```java
{package com.example.quickstart;

import io.github.seanchatmangpt.jotp.*;
import java.time.Duration;

/**
 * Your first JOTP application - A supervised counter process.
 */
public class QuickStart {

    // Define process state as an immutable record
    record Counter(int value) {}

    // Define messages using sealed interfaces
    sealed interface CounterMsg permits Increment, Get {}
    record Increment(int by) implements CounterMsg {}
    record Get() implements CounterMsg {}

    public static void main(String[] args) throws Exception {
        System.out.println("🚀 Starting JOTP Quick Start...");

        // Create a supervisor with ONE_FOR_ONE strategy
        var supervisor = new Supervisor(
            Supervisor.Strategy.ONE_FOR_ONE,
            5,  // max restarts
            Duration.ofMinutes(1)  // time window
        );

        // Create a supervised counter process
        ProcRef<Counter, CounterMsg> counter = supervisor.supervise(
            "counter",
            new Counter(0),  // initial state
            (state, msg) -> switch (msg) {
                case Increment(var by) -> {
                    System.out.println("📈 Incrementing by " + by);
                    yield new Counter(state.value() + by);
                }
                case Get() -> state;  // Return current state
            }
        );

        // Send some messages
        System.out.println("\n--- Sending Messages ---");
        counter.tell(new Increment(5));
        counter.tell(new Increment(3));
        counter.tell(new Increment(2));

        // Query the state
        Counter result = counter.ask(new Get(), Duration.ofSeconds(1)).get();
        System.out.println("\n✅ Final count: " + result.value());

        // Clean shutdown
        supervisor.shutdown();
        System.out.println("\n✨ Application shutdown complete!");
    }
}}
```

## Understanding the Core Concepts

### 1. Processes are Lightweight

Each JOTP process runs in its own virtual thread, using only ~1KB of heap memory. You can create millions of processes:

```java
// This creates 1 million processes - no problem!
for (int i = 0; i < 1_000_000; i++) {
    supervisor.supervise("proc-" + i, initialState, handler);
}
```

### 2. Messages are Immutable

We use sealed interfaces and records to ensure messages can't be modified:

```java
sealed interface CounterMsg permits Increment, Get {}
record Increment(int by) implements CounterMsg {}
```

This prevents shared state bugs and makes concurrency safe.

### 3. Let It Crash Philosophy

Notice there's no try-catch in our message handler. In JOTP:

- **Process crashes are expected** - they're handled by the supervisor
- **No defensive programming** - focus on the happy path
- **Automatic recovery** - supervisors restart crashed processes

## Try It: Make It Crash!

Let's see fault tolerance in action. Modify your `QuickStart.java`:

```java
{ProcRef<Counter, CounterMsg> counter = supervisor.supervise(
    "counter",
    new Counter(0),
    (state, msg) -> switch (msg) {
        case Increment(var by) -> {
            if (by < 0) {
                throw new RuntimeException("Negative increment!");
            }
            System.out.println("📈 Incrementing by " + by);
            yield new Counter(state.value() + by);
        }
        case Get() -> state;
    }
);

// This will crash the process
counter.tell(new Increment(-1));

// But the supervisor restarts it automatically!
Thread.sleep(100);  // Give it time to restart
counter.tell(new Increment(5));  // This still works

Counter result = counter.ask(new Get(), Duration.ofSeconds(1)).get();
System.out.println("✅ Count after crash: " + result.value());}
```
