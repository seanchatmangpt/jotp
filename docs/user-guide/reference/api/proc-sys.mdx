# ProcSys API Reference

**OTP `sys` module** — Process introspection, debug tracing, and hot code change without stopping the process.

## Overview

`ProcSys` provides runtime observability into running processes, matching OTP's `sys` module. You can inspect state, suspend/resume processing, collect statistics, enable tracing, and perform hot code upgrades — all without stopping the process.

### Key Capabilities

- **Live State Inspection**: `getState()` returns current state without modification
- **Suspend/Resume**: Pause and resume message processing for hot code upgrades
- **Statistics Tracking**: Monitor message throughput and queue depth
- **Debug Tracing**: Live event logging with custom formatters
- **Hot Code Upgrade**: Transform state atomically between messages

### OTP Equivalence

| Erlang/OTP | Java 26 |
|------------|---------|
| `sys:get_state(Pid)` | `ProcSys.getState(proc)` |
| `sys:suspend(Pid)` | `ProcSys.suspend(proc)` |
| `sys:resume(Pid)` | `ProcSys.resume(proc)` |
| `sys:statistics(Pid, get)` | `ProcSys.statistics(proc)` |
| `sys:trace(Pid, true)` | `ProcSys.trace(proc, true)` |
| `sys:get_log(Pid)` | `ProcSys.getLog(proc)` |
| `system_code_change/4` | `ProcSys.codeChange(proc, transformer)` |

## Public API

### State Inspection

#### `getState(Proc<S, M> proc)`

Asynchronously fetch the current state of `proc`. Enqueues a high-priority system request, ensuring a consistent snapshot taken between user messages.

```java
var proc = Proc.spawn(new CounterState(0), handler);
proc.tell(new Increment(5));
proc.tell(new Increment(3));

// Get current state asynchronously
CompletableFuture<CounterState> future = ProcSys.getState(proc);
CounterState state = future.get(1, TimeUnit.SECONDS);
System.out.println("Count: " + state.count()); // Output: 8
```

**Returns:** `CompletableFuture<S>` completing with the process state

**Behavior:**
- High-priority system message (processed before user messages)
- Consistent snapshot (taken between two user messages)
- Fails exceptionally if process has terminated
- Non-blocking for the caller

---

#### `suspend(Proc<?, ?> proc)`

Pause message processing. Messages continue to accumulate in the mailbox but are not processed until `resume()` is called.

```java
ProcSys.suspend(proc);

// Send messages (they queue up)
proc.tell(new WorkItem1());
proc.tell(new WorkItem2());

// State hasn't changed yet
CounterState state = ProcSys.getState(proc).get();

// Resume processing
ProcSys.resume(proc);
```

**Use cases:**
- Hot code upgrades (suspend → swap code → resume)
- Debugging (freeze state to inspect)
- Testing (pause to simulate slow processing)

---

#### `resume(Proc<?, ?> proc)`

Resume message processing after a `suspend()`.

```java
ProcSys.suspend(proc);
// ... do work ...
ProcSys.resume(proc);
// Messages are now processed
```

---

### Statistics

#### `statistics(Proc<?, ?> proc)`

Get a point-in-time statistics snapshot.

```java
ProcSys.Stats stats = ProcSys.statistics(proc);
System.out.println("Messages in: " + stats.messagesIn());
System.out.println("Messages out: " + stats.messagesOut());
System.out.println("Queue depth: " + stats.queueDepth());
```

**Returns:** `Stats` record with:
- `messagesIn()` — Total messages received since process start
- `messagesOut()` — Total messages processed since process start
- `queueDepth()` — Current number of messages in mailbox

**Behavior:**
- Reads atomic counters (non-transactional but accurate within 1 message cycle)
- No blocking (direct counter reads)
- Useful for monitoring and alerting

---

### Debug Tracing

#### `trace(Proc<S, M> proc, boolean enable)`

Enable or disable live event tracing with default stdout formatter.

```java
// Enable tracing
ProcSys.trace(proc, true);

// All messages are now printed to stdout:
// proc event = In(Increment(5))
// proc event = Out(Increment(5), CounterState(5))

// Disable tracing
ProcSys.trace(proc, false);
```

**Behavior:**
- Prints each message received and processed
- Format: `"proc event = <event>"`
- Retains up to 100 recent events for `getLog()`
- Matches OTP's `write_debug/3` convention

---

#### `trace(Proc<S, M> proc, boolean enable, DebugFormatter<M> formatter)`

Enable tracing with a custom formatter.

```java
// Custom formatter with timestamps
DebugFormatter<MyMsg> formatter = (out, event, procName) -> {
    String timestamp = DateTimeFormatter.ISO_INSTANT.format(Instant.now());
    out.printf("[%s] %s: %s%n", timestamp, procName, event);
};

ProcSys.trace(proc, true, formatter);
```

---

#### `trace(Proc<S, M> proc, boolean enable, DebugFormatter<M> formatter, PrintWriter out, Object procName)`

Enable tracing with custom output and process name.

```java
// Trace to file
var fileWriter = new PrintWriter(new FileWriter("trace.log"), true);
ProcSys.trace(proc, true, formatter, fileWriter, "my-worker");
```

---

#### `getLog(Proc<S, M> proc)`

Retrieve the event log from the trace observer.

```java
ProcSys.trace(proc, true, formatter);

// ... send messages ...

List<DebugEvent<MyMsg>> log = ProcSys.getLog(proc);
for (DebugEvent<MyMsg> event : log) {
    System.out.println("Event: " + event);
}
```

**Returns:** Immutable list of recent events (oldest first, capped at 100)

---

### Hot Code Upgrade

#### `codeChange(Proc<S, M> proc, Function<S, S> transformer)`

Apply a state-transformation function to a running process atomically between messages.

```java
// Migrate from v1 to v2 state schema
record OldStateV1(String field1) {}
record NewStateV2(String field1, int field2) {}

var newState = ProcSys.codeChange(proc, old -> new NewStateV2(old.field1(), 0));
```

**Returns:** The new state after transformation

**Throws:**
- `InterruptedException` if the calling thread is interrupted
- `IllegalStateException` if the process has terminated or transformation threw

**Behavior:**
- Transformation applied at next sys-drain checkpoint (before user message)
- Process never paused (transformation is atomic)
- Mirrors OTP's `system_code_change/4` protocol

---

### Debug Options (Process-Internal)

#### `debugOptions()`, `debugOptions(boolean trace)`, `debugOptions(boolean trace, int maxLog)`

Create `DebugOptions` for process-internal debug pipelines.

```java
// No tracing, no log
DebugOptions<MyMsg> deb = ProcSys.debugOptions();

// Trace enabled, no log
DebugOptions<MyMsg> deb = ProcSys.debugOptions(true);

// Trace and log (retain last 50 events)
DebugOptions<MyMsg> deb = ProcSys.debugOptions(true, 50);
```

---

#### `handleDebug(DebugOptions<M> deb, DebugFormatter<M> formatter, Object info, DebugEvent<M> event)`

Record a debug event and optionally print it. For use in custom/special processes.

```java
// In a custom process handler
DebugOptions<MyMsg> deb = ProcSys.debugOptions(true, 100);

// Record an event
deb = ProcSys.handleDebug(deb, formatter, "my-process", new DebugEvent.In<>(msg));
```

**Returns:** Updated `DebugOptions` with event recorded

---

## Implementation Details

### System Message Protocol

All `getState` and `codeChange` operations enqueue a `SysRequest` into the process's high-priority sys channel:

```java
public static <S, M> CompletableFuture<S> getState(Proc<S, M> proc) {
    if (!proc.thread().isAlive()) {
        return CompletableFuture.failedFuture(new IllegalStateException("process is not alive"));
    }
    var future = new CompletableFuture<Object>();
    proc.offerSysRequest(new SysRequest.GetState(future));
    return future.thenApply(s -> (S) s);
}
```

**Protocol:**
1. Caller enqueues `SysRequest` to process's sys channel
2. Process drains sys channel before each user message
3. Process handles request (e.g., capture state, apply transformation)
4. Process completes the future with result

**Benefits:**
- No direct field access (process handles its own system messages)
- Consistent snapshots (taken between user messages)
- Matches OTP's `{system, From, Request}` protocol

### Debug Observer Pattern

Tracing uses the observer pattern:

```java
// Attach observer
proc.setDebugObserver(new DefaultDebugObserver<>(formatter, out, procName, 100));

// Process calls observer on each message
observer.onIn(msg);   // When message is received
observer.onOut(state, msg);  // After message is processed
```

**Thread Safety:**
- Observer called from process virtual thread only
- Log is synchronized (ring buffer accessed from process thread and `getLog()`)
- Detach is atomic (null assignment)

### Suspend/Resume Implementation

Suspend sets a flag checked by the message loop:

```java
public void suspendProc() {
    suspended.set(true);
}

public void resumeProc() {
    suspended.set(false);
}

// In process loop:
if (suspended.get()) {
    Thread.sleep(50);  // Wait for resume
    continue;  // Re-check suspend flag
}
```

**Behavior:**
- Messages continue to queue in mailbox
- Process polls suspend flag every 50ms
- Low overhead (simple volatile flag read)

## Common Usage Patterns

### Runtime State Inspection

```java
// Monitor process state in production
var state = ProcSys.getState(proc).get(5, TimeUnit.SECONDS);
metrics.gauge("process.state.value", state.value());
```

### Hot Code Upgrade

```java
// Upgrade state schema without downtime
var oldProc = Proc.spawn(new OldState("data"), oldHandler);

// Migrate to new state
var newState = ProcSys.codeChange(oldProc, old ->
    new NewState(old.field1(), 0, "default")
);

// Swap handler
oldProc.setHandler(newHandler);
```

### Debugging Production Issues

```java
// Enable tracing on a misbehaving process
ProcSys.trace(proc, true);

// Wait for issue to occur
Thread.sleep(10000);

// Retrieve log
var log = ProcSys.getLog(proc);
logger.info("Debug log: {}", log);

// Disable tracing
ProcSys.trace(proc, false);
```

### Performance Monitoring

```java
// Periodic statistics collection
ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);
scheduler.scheduleAtFixedRate(() -> {
    var stats = ProcSys.statistics(proc);
    metrics.gauge("proc.messages.in", stats.messagesIn());
    metrics.gauge("proc.messages.out", stats.messagesOut());
    metrics.gauge("proc.queue.depth", stats.queueDepth());
}, 0, 10, TimeUnit.SECONDS);
```

### Suspended Debugging

```java
// Freeze process to inspect state
ProcSys.suspend(proc);

// Inspect state
var state = ProcSys.getState(proc).get();
System.out.println("Frozen state: " + state);

// Inspect mailbox
int queueDepth = ProcSys.statistics(proc).queueDepth();

// Resume when done
ProcSys.resume(proc);
```

## Best Practices

### 1. Always Use Timeouts on getState()

```java
// Good: Explicit timeout
var state = ProcSys.getState(proc)
    .get(5, TimeUnit.SECONDS);

// Bad: No timeout (potential deadlock)
var state = ProcSys.getState(proc).get(); // May block forever
```

### 2. Disable Tracing in Production (Unless Debugging)

```java
// Good: Tracing off by default
ProcSys.trace(proc, false);

// Enable temporarily for debugging
ProcSys.trace(proc, true, formatter);
// ... debug ...
ProcSys.trace(proc, false);
```

### 3. Use Custom Formatters for Better Logs

```java
// Good: Structured logging
DebugFormatter<MyMsg> formatter = (out, event, procName) -> {
    out.printf("{\"proc\":\"%s\",\"event\":\"%s\",\"time\":%d}%n",
        procName, event, System.currentTimeMillis());
};

// Bad: Default formatter (not machine-parseable)
ProcSys.trace(proc, true);
```

### 4. Suspend Briefly (Avoid Message Accumulation)

```java
// Good: Brief suspend for state inspection
ProcSys.suspend(proc);
var state = ProcSys.getState(proc).get();
ProcSys.resume(proc);

// Bad: Long suspend (mailbox grows)
ProcSys.suspend(proc);
Thread.sleep(60000);  // Mailbox may fill up!
ProcSys.resume(proc);
```

### 5. Use Statistics for Monitoring, Not Control Flow

```java
// Good: Monitoring and alerting
var stats = ProcSys.statistics(proc);
if (stats.queueDepth() > 1000) {
    logger.warn("Mailbox backlog: {}", stats.queueDepth());
}

// Bad: Using statistics for control flow
while (ProcSys.statistics(proc).queueDepth() > 0) {
    Thread.sleep(10);  // Busy waiting!
}
```

## Gotchas

### 1. getState() Can See Inconsistent State

```java
// Risk: State captured mid-transition
proc.tell(new Increment(1));
proc.tell(new Increment(1));
var state = ProcSys.getState(proc).get(); // Might be 0, 1, or 2
```

**Note:** State is always consistent at a single point in time, but you might capture it before or after specific messages.

### 2. Suspend Doesn't Prevent Crashes

```java
// Risk: Process can still crash while suspended
ProcSys.suspend(proc);
proc.tell(new CrashMessage());  // Will crash when resumed!
```

**Behavior:** Suspend blocks message processing, but crash callbacks still fire when the crashing message is eventually processed.

### 3. Trace Log is Bounded

```java
// Risk: Old events are dropped
ProcSys.trace(proc, true, formatter, out, "proc", 100);

// Send 200 messages
for (int i = 0; i < 200; i++) {
    proc.tell(new WorkItem(i));
}

var log = ProcSys.getLog(proc);
System.out.println(log.size()); // Output: 100 (only recent 100)
```

### 4. Hot Code Upgrade Must Preserve Type

```java
// Risk: Type mismatch after upgrade
record OldState(int value) {}
record NewState(String value) {}

// This will fail at runtime!
ProcSys.codeChange(proc, old -> new NewState(String.valueOf(old.value())));
// Proc<S, M> still expects S = OldState, but we returned NewState
```

**Solution:** Use compatible types or re-create the process:

```java
// For incompatible types, recreate process
var oldState = ProcSys.getState(proc).get();
var newState = migrate(oldState);
var newProc = Proc.spawn(newState, newHandler);
```

## Performance Considerations

### Operation Latency

| Operation | Latency | Notes |
|-----------|---------|-------|
| `getState()` | <1μs enqueue + processing | Future completion |
| `suspend()` | <100ns | Volatile flag write |
| `resume()` | <100ns | Volatile flag write |
| `statistics()` | ~200ns | Three atomic counter reads |
| `trace()` | ~100ns | Observer assignment |

### Tracing Overhead

- **Enabled**: ~5-10μs per message (observer calls)
- **Disabled**: ~0ns (null check only)

### Memory Footprint

- **Trace log**: ~100 events × 32 bytes = ~3.2 KB per traced process
- **Debug observer**: ~100 bytes per traced process

## Production Usage

### Health Check Endpoints

```java
@GetMapping("/health/proc/{name}")
public ResponseEntity<Map<String, Object>> health(@PathVariable String name) {
    return ProcRegistry.whereis(name)
        .map(proc -> {
            try {
                var state = ProcSys.getState(proc).get(1, TimeUnit.SECONDS);
                var stats = ProcSys.statistics(proc);
                return ResponseEntity.ok(Map.of(
                    "alive", true,
                    "state", state.toString(),
                    "queueDepth", stats.queueDepth()
                ));
            } catch (Exception e) {
                return ResponseEntity.status(503).body(Map.of("error", e.getMessage()));
            }
        })
        .orElse(ResponseEntity.notFound().build());
}
```

### Debug Console

```java
// Admin endpoint for runtime debugging
@PostMapping("/admin/debug/trace")
public String enableTrace(@RequestParam String procName) {
    return ProcRegistry.whereis(procName)
        .map(proc -> {
            ProcSys.trace(proc, true, DebugFormatter.defaultFormatter());
            return "Tracing enabled for " + procName;
        })
        .orElse("Process not found: " + procName);
}
```
