# ProcTimer API Reference

**OTP-style process timers** — `timer:send_after/3` and `timer:send_interval/3`.

## Overview

`ProcTimer` delivers messages to processes after delays or at intervals. Unlike blocking sleep or callback-based APIs, timers deliver messages that the process handles in its main receive loop.

### Key Characteristics

- **Message-Based** — Timers deliver messages to mailboxes
- **One-Shot** — `sendAfter()` delivers a single message
- **Repeating** — `sendInterval()` delivers messages periodically
- **Cancellable** — Return `TimerRef` to cancel before delivery
- **Shared Scheduler** — Single daemon thread manages all timers

### OTP Equivalence

| Erlang/OTP | Java 26 |
|------------|---------|
| `timer:send_after(Ms, Pid, Msg)` | `ProcTimer.sendAfter(ms, proc, msg)` |
| `timer:send_interval(Ms, Pid, Msg)` | `ProcTimer.sendInterval(ms, proc, msg)` |
| `timer:cancel(TRef)` | `ProcTimer.cancel(ref)` or `ref.cancel()` |

## Public API

### One-Shot Timers

#### `sendAfter(long delayMs, Proc<?, M> target, M msg)`

Send `msg` to `target` after `delayMs` milliseconds.

```java
Proc<String, String> proc = Proc.spawn(state, handler);

TimerRef timer = ProcTimer.sendAfter(1000, proc, "timeout");

// After 1 second, proc receives "timeout" message
```

**Returns:** `TimerRef` for cancellation

---

#### `sendAfter(Duration delay, Proc<?, M> target, M msg)`

Duration-based overload.

```java
TimerRef timer = ProcTimer.sendAfter(
    Duration.ofSeconds(5),
    proc,
    "wake-up"
);
```

---

### Repeating Timers

#### `sendInterval(long periodMs, Proc<?, M> target, M msg)`

Send `msg` to `target` every `periodMs` milliseconds.

```java
TimerRef ticker = ProcTimer.sendInterval(1000, proc, "tick");

// proc receives "tick" every second until canceled
```

**First delivery:** After one full period

---

#### `sendInterval(Duration interval, Proc<?, M> target, M msg)`

Duration-based overload.

```java
TimerRef ticker = ProcTimer.sendInterval(
    Duration.ofMinutes(1),
    proc,
    "heartbeat"
);
```

---

### Cancellation

#### `cancel(TimerRef ref)`

Cancel a timer.

```java
TimerRef timer = ProcTimer.sendAfter(5000, proc, "timeout");

// Later: cancel before delivery
boolean canceled = ProcTimer.cancel(timer);
if (canceled) {
    System.out.println("Timer canceled");
}
```

**Returns:** `true` if timer was still pending

---

#### `TimerRef.cancel()`

Instance method for cancellation.

```java
TimerRef timer = ProcTimer.sendAfter(5000, proc, "timeout");
timer.cancel();  // Same as ProcTimer.cancel(timer)
```

## Usage Patterns

### Request Timeout

```java
record Request(String data, CompletableFuture<String> reply) {}
record Timeout() {}

Proc<RequestState, Object> server = Proc.spawn(state, (s, m) -> switch (m) {
    case Request(var data, var reply) -> {
        // Start timeout
        ProcTimer.sendAfter(5000, server.proc(), new Timeout());
        // Process request...
        yield s;
    }
    case Timeout() -> {
        // Handle timeout
        System.err.println("Request timed out");
        yield s;
    }
    default -> yield s;
});
```

---

### Periodic Heartbeat

```java
sealed interface Msg permits Start, Stop, Heartbeat {}
record Start() implements Msg {}
record Stop() implements Msg {}
record Heartbeat() implements Msg {}

Proc<HeartbeatState, Msg> proc = Proc.spawn(state, (s, m) -> switch (m) {
    case Start() -> {
        // Start heartbeat timer
        var timer = ProcTimer.sendInterval(1000, proc.proc(), new Heartbeat());
        yield s.withTimer(timer);
    }
    case Stop() -> {
        // Stop heartbeat timer
        s.timer().cancel();
        yield s.withTimer(null);
    }
    case Heartbeat() -> {
        System.out.println("Heartbeat: " + Instant.now());
        yield s;
    }
});
```

---

### Delayed Retry

```java
sealed interface Msg permits Process, Retry {}
record Process(String data) implements Msg {}
record Retry(String data) implements Msg {}

Proc<RetryState, Msg> proc = Proc.spawn(state, (s, m) -> switch (m) {
    case Process(var data) -> {
        try {
            // Try to process
            externalService.call(data);
            yield s;  // Success
        } catch (Exception e) {
            // Retry after 1 second
            ProcTimer.sendAfter(1000, proc.proc(), new Retry(data));
            yield s;
        }
    }
    case Retry(var data) -> {
        // Retry attempt
        System.out.println("Retrying: " + data);
        yield s;
    }
});
```

---

### Debouncing

```java
record Input(String text) {}
record Save(String text) {}

Proc<EditorState, Object> editor = Proc.spawn(state, (s, m) -> switch (m) {
    case Input(var text) -> {
        // Cancel previous save timer
        if (s.saveTimer() != null) {
            s.saveTimer().cancel();
        }

        // Schedule new save after 2 seconds of inactivity
        var timer = ProcTimer.sendAfter(2000, editor.proc(), new Save(text));
        yield s.withSaveTimer(timer);
    }
    case Save(var text) -> {
        System.out.println("Saving: " + text);
        fileSystem.write(text);
        yield s.withSaveTimer(null);
    }
    default -> yield s;
});
```

---

### Scheduled Task

```java
sealed interface Msg permits Schedule, Run {}
record Schedule(Instant when, Runnable task) implements Msg {}
record Run(Runnable task) implements Msg {}

Proc<SchedulerState, Msg> scheduler = Proc.spawn(state, (s, m) -> switch (m) {
    case Schedule(var when, var task) -> {
        var delay = Duration.between(Instant.now(), when).toMillis();
        if (delay > 0) {
            ProcTimer.sendAfter(delay, scheduler.proc(), new Run(task));
        }
        yield s;
    }
    case Run(var task) -> {
        task.run();
        yield s;
    }
});

// Schedule task 5 minutes from now
scheduler.tell(new Schedule(
    Instant.now().plus(Duration.ofMinutes(5)),
    () -> System.out.println("Executed!")
));
```

## Implementation Details

### Shared Scheduler

All timers use a single `ScheduledExecutorService` with one daemon platform thread:

```java
private static final ScheduledExecutorService SCHEDULER =
    Executors.newScheduledThreadPool(1, r -> {
        Thread t = new Thread(r, "proc-timer");
        t.setDaemon(true);
        return t;
    });
```

**Why Platform Thread?**
- Timer wheel management is CPU-bound (not I/O)
- Virtual threads excel at I/O, not CPU work
- Single thread ensures fairness and minimal overhead
- Actual callback is non-blocking `Proc.tell()` enqueue

---

### Timer Ref

`TimerRef` wraps a `ScheduledFuture`:

```java
public record TimerRef(ScheduledFuture<?> future) {
    public boolean cancel() {
        return future.cancel(false);
    }
}
```

The `future.cancel(false)` parameter means:
- **Don't interrupt** if timer is currently firing
- Just prevent future executions

## Performance Characteristics

| Operation | Latency | Notes |
|-----------|---------|-------|
| `sendAfter()` | ~1μs | Schedule delay task |
| `sendInterval()` | ~1μs | Schedule recurring task |
| `cancel()` | ~100ns | Future.cancel() call |
| Timer firing | ~50ns | `Proc.tell()` enqueue |

**Scheduler Overhead:**
- Single daemon thread (minimal CPU)
- Timer wheel algorithm (O(log n) scheduling)
- ~1KB per scheduled timer

**Scalability:**
- Millions of timers supported
- Limited only by memory (~1 KB per timer)

## Best Practices

### 1. Always Store TimerRef for Cancellation

```java
// Good: Can cancel later
var timer = ProcTimer.sendAfter(5000, proc, msg);
// Later: timer.cancel();

// Bad: No way to cancel
ProcTimer.sendAfter(5000, proc, msg);  // TimerRef lost
```

---

### 2. Use Duration Overloads for Readability

```java
// Good: Clear duration
ProcTimer.sendAfter(Duration.ofSeconds(5), proc, msg);

// Less clear: Millisecond math
ProcTimer.sendAfter(5000, proc, msg);
```

---

### 3. Handle Timer Messages Explicitly

```java
sealed interface Msg permits Data, Timeout {}
record Data(String value) implements Msg {}
record Timeout() implements Msg {}

case Timeout() -> {
    // Explicitly handle timeout
    System.out.println("Operation timed out");
    yield state;
}
```

---

### 4. Cancel Timers on Process Stop

```java
record State(TimerRef timer) {}

case Stop() -> {
    if (s.timer() != null) {
        s.timer().cancel();
    }
    proc.stop();
    yield s;
}
```

## Gotchas

### 1. Interval Timers Continue Forever

```java
var ticker = ProcTimer.sendInterval(1000, proc, "tick");

// If you don't cancel, this runs forever
// Even if proc stops, the timer still fires (messages are dropped)

// Solution: Cancel explicitly
ticker.cancel();
```

---

### 2. Timer Messages Are Regular Messages

```java
ProcTimer.sendAfter(1000, proc, "timeout");

// "timeout" is just a message like any other
// It goes in the mailbox behind other messages
proc.tell("msg1");
proc.tell("msg2");
// "timeout" processed after msg1 and msg2
```

---

### 3. Cancel After Delivery is No-Op

```java
var timer = ProcTimer.sendAfter(100, proc, msg);
Thread.sleep(200);  // Timer has fired
timer.cancel();  // Returns false (already delivered)
```

---

### 4. First Interval Delivery After Period

```java
ProcTimer.sendInterval(1000, proc, "tick");

// First "tick" arrives after 1 second, not immediately
// Then every 1 second thereafter
```
