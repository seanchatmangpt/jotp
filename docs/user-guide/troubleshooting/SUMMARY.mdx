## Issue Categories

### 1. Build Issues

**When to use:** Compilation fails, IDE shows errors, Maven build problems

**Common problems:**
- Java 26 preview feature errors
- Maven compilation failures
- Spotless formatting issues
- Dependency conflicts
- Module system errors

**Quick fix:**
```bash
mvnd clean compile
mvnd spotless:apply
java -version  # Verify Java 26
```

---

### 2. Runtime Issues

**When to use:** Processes crash, messages timeout, system hangs

**Common problems:**
- Process won't start
- Messages not delivered
- Mailbox overflow
- Deadlock diagnosis
- Virtual thread pinning

**Quick fix:**
```java
// Enable debug tracing
DebugOptions options = DebugOptions.builder()
    .traceAll(true)
    .build();
Proc.setGlobalDebugOptions(options);

// Check process state
ProcInspection inspect = ProcSys.inspect(procId);
System.out.println("State: " + inspect.state());
System.out.println("Mailbox: " + inspect.mailboxSize());
```

---

### 3. Supervision Issues

**When to use:** Supervision tree not working, processes not restarting

**Common problems:**
- Children not restarting
- Supervisor crashes
- Restart intensity exceeded
- Orphaned processes
- Shutdown hangs

**Quick fix:**
```java
// Check supervisor state
SupervisorRef ref = supervisor.ref();
ref.children().forEach((id, info) -> {
    System.out.println(id + ": " + info.state());
    System.out.println("  Restarts: " + info.restartCount());
});

// Verify restart strategy
System.out.println("Strategy: " + ref.strategy());
```

---

### 4. Performance Issues

**When to use:** System slow, high memory usage, poor throughput

**Common problems:**
- High message latency
- Memory leaks
- GC pressure
- Thread exhaustion
- Slow startups

**Quick fix:**
```bash
# Profile with JFR
java -XX:StartFlightRecording=filename=rec.jfr,duration=60s ...

# Check thread counts
jcmd <pid> Thread.vthread_summary

# Heap dump
jmap -dump:format=b,file=heap.hprof <pid>
```

---

### 5. Testing Issues

**When to use:** Tests fail intermittently, race conditions

**Common problems:**
- Flaky async tests
- Awaitility timeouts
- Mock setup problems
- Test isolation failures

**Quick fix:**
```java
// Use Awaitility properly
await()
    .atMost(5, TimeUnit.SECONDS)
    .pollInterval(100, TimeUnit.MILLISECONDS)
    .untilAsserted(() -> {
        assertEquals(expected, actual);
    });

// Clean up after tests
@AfterEach
void cleanup() {
    ProcRegistry.getInstance().clear();
}
```

---

### 6. Production Issues

**When to use:** Production debugging, distributed system problems

**Common problems:**
- Production monitoring setup
- Distributed system issues
- Memory issues in production
- Performance degradation over time

**Quick fix:**
```bash
# Production health check
curl http://localhost:8080/health
curl http://localhost:8080/metrics

# Safe diagnostics
jcmd <pid> ProcSys.summary
jcmd <pid> Thread.print_to_file -file=/tmp/threads.txt

# Enable tracing temporarily
jcmd <pid> ProcSys.setTraceLevel VERBOSE
# Remember: jcmd <pid> ProcSys.setTraceLevel WARN
```

## Common Error Quick Reference

| Error | Quick Fix | Guide |
|-------|-----------|-------|
| `ClassCastException` in handler | Use sealed interfaces with exhaustive matching | [Common Errors](/docs/user-guide/troubleshooting/common-errors) |
| Process already registered | Check registry before registering | [Common Errors](/docs/user-guide/troubleshooting/common-errors) |
| `TimeoutException` in ask() | Always use timeouts with fallback | [Common Errors](/docs/user-guide/troubleshooting/common-errors) |
| Mailbox overflow | Implement backpressure or increase capacity | [Runtime Issues](/docs/user-guide/troubleshooting/runtime-issues) |
| Children not restarting | Check restart intensity and strategy | [Supervision Issues](/docs/user-guide/troubleshooting/supervision-issues) |
| OutOfMemoryError | Capture heap dump, analyze for leaks | [Performance Issues](/docs/user-guide/troubleshooting/performance-issues) |
| Flaky test | Use Awaitility with proper waits | [Testing Issues](/docs/user-guide/troubleshooting/testing-issues) |
| Virtual thread pinning | Replace synchronized with ReentrantLock | [Runtime Issues](/docs/user-guide/troubleshooting/runtime-issues) |

## Diagnostic Workflow

### Step 1: Identify the Issue Category

Use the flowchart in the [Diagnostic Checklist](/docs/user-guide/troubleshooting/diagnostic-checklist) to identify your issue category.

### Step 2: Enable Debugging

```java
// Enable debug tracing
DebugOptions options = DebugOptions.builder()
    .traceAll(true)
    .build();
Proc.setGlobalDebugOptions(options);
```

### Step 3: Collect Diagnostic Information

```bash
# Thread dump
jcmd <pid> Thread.print_to_file -file=threads.txt

# Heap dump (if memory issue)
jmap -dump:format=b,file=heap.hprof <pid>

# JFR recording
java -XX:StartFlightRecording=filename=rec.jfr,duration=60s ...
```

### Step 4: Follow the Relevant Guide

Navigate to the specific guide for your issue category and follow the systematic diagnosis procedures.

### Step 5: Apply Solutions

Try the solutions in order of likelihood. Start with "Quick Fix" sections, then work through detailed solutions.

### Step 6: Verify and Monitor

```bash
# Check system health
jcmd <pid> ProcSys.summary

# Monitor metrics
curl http://localhost:8080/metrics
```

## Key Diagnostic Commands

### Process Inspection

```java
// Check process state
ProcInspection inspect = ProcSys.inspect(procId);
System.out.println("State: " + inspect.state());
System.out.println("Mailbox: " + inspect.mailboxSize());
System.out.println("Alive: " + inspect.isAlive());

// List all processes
Set<ProcId> all = ProcSys.allProcesses();
```

### JVM Diagnostics

```bash
# Thread dump
jcmd <pid> Thread.print_to_file -file=threads.txt

# Heap info
jcmd <pid> GC.heap_info

# Virtual threads
jcmd <pid> Thread.vthread_summary

# GC stats
jstat -gcutil <pid> 1000 10
```

### JOTP-Specific Commands

```bash
# Process summary
jcmd <pid> ProcSys.summary

# List processes
jcmd <pid> ProcSys.listProcesses

# Set trace level
jcmd <pid> ProcSys.setTraceLevel VERBOSE

# Check version
jcmd <pid> ProcSys.version
```

## Prevention Best Practices

### Development

- [ ] Always use sealed interfaces for messages
- [ ] Enable exhaustive pattern matching
- [ ] Handle exceptions in all handlers
- [ ] Use timeouts on `ask()` calls
- [ ] Enable debug tracing during development

### Testing

- [ ] Use Awaitility for async assertions
- [ ] Clean up processes in `@AfterEach`
- [ ] Run tests multiple times to detect flakiness
- [ ] Test with realistic load
- [ ] Verify process cleanup

### Production

- [ ] Set up monitoring and alerting
- [ ] Implement health checks
- [ ] Use bounded mailboxes
- [ ] Enable distributed tracing
- [ ] Regular heap dump analysis in staging
- [ ] Monitor GC pause times
- [ ] Profile before deploying

### Before Asking for Help

1. **Read the relevant guide** for your issue category
2. **Follow the diagnostic checklist** to gather information
3. **Enable debug tracing** and collect logs
4. **Search existing issues** in the repository
