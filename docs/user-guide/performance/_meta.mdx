## Performance Best Practices

### 1. Start with Baseline Measurements

Before optimizing, measure your baseline:

```java
// Use JMH for microbenchmarks
@Benchmark
public void tell_throughput() {
    countingActor.tell(1);
}

// Use stress tests for load testing
MetricsCollector metrics = runStressTest(
    "Constant Load (10K msg/sec)",
    new LoadProfile.ConstantLoad(10_000L, Duration.ofSeconds(10)),
    () -> proc.tell("message")
);
```

### 2. Monitor Continuously

Set up comprehensive monitoring before going live:

```java
// Core metrics
MeterRegistry registry = new PrometheusMeterRegistry(PrometheusConfig.DEFAULT);
Gauge.builder("jotp_mailbox_depth", proc, p -> p.getMailboxSize())
    .register(registry);

// Prometheus scraping
// http://localhost:9095/metrics
```

### 3. Define SLOs Early

Establish service level objectives:

```yaml
SLOs:
  - 99.9% of messages delivered within 1ms (P99)
  - 99.95% of processes available per month
  - <5 minutes to detect and respond to incidents
  - <0.1% message drop rate
```

### 4. Test Failures

Validate fault tolerance:

```java
// Simulate crashes
ProcRef<String, Integer> worker = supervisor.supervise("worker", initialState, handler);
worker.tell(new CrashMessage());  // Force crash

// Verify restart
ProcStatistics stats = ProcSys.of(worker).getStatistics();
assertEquals(1, stats.restartCount());
```

### 5. Profile Regularly

Use profiling tools:

```bash
# JFR profiling
java -XX:+FlightRecorder -XX:StartFlightRecording=duration=60s -jar app.jar

# Analyze with JDK Mission Control
# Look for hot methods, GC pauses, thread contention
```
