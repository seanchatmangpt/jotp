# Fault Tolerance - Crash Recovery & Supervision Across Nodes

import { Diagram } from '@/components/diagram';

## Overview

JOTP's fault tolerance extends the OTP "let it crash" philosophy across distributed nodes, providing automatic restart, supervised recovery, and split-brain resolution — all without manual intervention.

## The "Let It Crash" Philosophy

**Joe Armstrong:** "In a reliable system, you don't try to prevent errors — you design for them. Processes crash. Supervisors restart them. The system heals itself."

### Core Principles

1. **Failure Is Normal** — Crashes are expected, not exceptional
2. **Isolation** — Each process has a crash boundary
3. **Supervision** — Supervisors monitor and restart children
4. **Transparency** - Crashes are contained; the system continues

## Supervision Trees

Supervisor supervisor = Supervisor.builder()

    .timeWindow(Duration.ofSeconds(10))
    .build();

// Spawn supervised children
ProcRef<PdaState, PdaMsg> vCar = supervisor.supervise(

## Restart Strategies

### ONE_FOR_ONE

```java
Supervisor.builder()
    .strategy(Supervisor.Strategy.ONE_FOR_ONE)
    .build();

// Behavior: Only the crashed child restarts
// Use case: Independent children (no shared state)
// Example: Parameter data access processes
```

### ONE_FOR_ALL

```java
Supervisor.builder()
    .strategy(Supervisor.Strategy.ONE_FOR_ALL)
    .build();

// Behavior: All children restart when one crashes
// Use case: Tightly coupled children with shared dependencies
// Example: Session manager + parameter registry
```

### REST_FOR_ONE

```java
Supervisor.builder()
    .strategy(Supervisor.Strategy.REST_FOR_ONE)
    .build();

// Behavior: Crashed child + all children started after it restart
// Use case: Ordered startup with dependencies
// Example: Connection pool → query executor → result processor
```

## Distributed Failure Detection

### Heartbeat Protocol

```java
// Cluster membership actor sends heartbeats
sealed interface ClusterMsg permits
    Heartbeat, HeartbeatAck, GossipState, MemberSuspected, MemberDown {}

// Heartbeat loop (runs on each node)
void heartbeatLoop() {
    while (running) {
        for (NodeId peer : clusterMembers) {
            if (peer.equals(self.nodeId())) continue;

            // Send heartbeat
            send(peer, new Heartbeat(self.nodeId(), Instant.now()));

            // Wait for ACK
            CompletableFuture<HeartbeatAck> ack = expectAck(peer, Duration.ofMillis(500));

            if (ack == null) {
                // Heartbeat missed — suspect node
                handleSuspectedPeer(peer);
            }
        }

        Thread.sleep(500); // Heartbeat interval
    }
}
```

### Phi Accrual Failure Detector

```java
// Adaptive failure detection (Erlang-style)
class PhiAccrualDetector {
    private final double threshold;
    private final List<Long> interArrivalTimes = new ArrayList<>();

    public double phi(NodeId node) {
        long now = Instant.now().toEpochMilli();
        long lastSeen = lastHeartbeat(node);

        // Calculate phi based on inter-arrival time distribution
        double mean = mean(interArrivalTimes);
        double stdDev = stdDev(interArrivalTimes);

        // Phi increases as time since last heartbeat increases
        return (now - lastSeen - mean) / stdDev;
    }

    public boolean isSuspected(NodeId node) {
        return phi(node) > threshold; // Default: 8.0
    }
}
```

### Oldest-Node-Wins Strategy

```java
// Each node knows its start epoch (milliseconds since Unix epoch)
record NodeId(String name, String host, int port, UUID instanceId, long startEpoch) {}

// Split-brain resolution
class OldestNodeWinsSplitBrainStrategy implements SplitBrainStrategy {
    @Override
    public boolean shouldSurvive(Set<NodeId> localPartition, Set<NodeId> allKnownNodes) {
        // Find the oldest node in the entire cluster
        NodeId oldest = allKnownNodes.stream()
            .min(Comparator.comparingLong(NodeId::startEpoch))
            .orElseThrow();

        // Survive only if this partition contains the oldest node
        return localPartition.contains(oldest);
    }
}

// Usage
ClusterSupervisor supervisor = new ClusterSupervisor(
    new OldestNodeWinsSplitBrainStrategy()
);

// When partition heals:
// - Partition without oldest node shuts down
// - Oldest node's partition continues as the authoritative cluster
```

### Alternative Strategies

```java
// Quorum-based survival
class QuorumSplitBrainStrategy implements SplitBrainStrategy {
    private final int minNodes;

    @Override
    public boolean shouldSurvive(Set<NodeId> localPartition, Set<NodeId> allKnownNodes) {
        return localPartition.size() >= minNodes;
    }
}

// Static seed node survival
class StaticSeedSplitBrainStrategy implements SplitBrainStrategy {
    private final Set<NodeId> seedNodes;

    @Override
    public boolean shouldSurvive(Set<NodeId> localPartition, Set<NodeId> allKnownNodes) {
        // Survive if partition contains at least one seed node
        return localPartition.stream().anyMatch(seedNodes::contains);
    }
}
```

## Crash Recovery Patterns

### Transient Error Recovery

```java
// Retry with exponential backoff
CrashRecovery.retry(3, () -> {
    return database.query("SELECT * FROM inventory WHERE sku = ?", sku);
});
```

### Database Connection Recovery

```java
// Supervised database connection process
Proc<ConnectionState, DbMsg> dbProc = Proc.start(
    initialState -> msg -> {
        return switch (msg) {
            case Query(var sql, var params, var reply) -> {
                try {
                    ResultSet rs = state.connection().query(sql, params);
                    reply.complete(Result.ok(rs));
                    yield state;
                } catch (SQLException e) {
                    // Connection broken — crash and let supervisor restart
                    throw new RuntimeException("Database connection failed", e);
                }
            }
        };
    },
    new ConnectionState(connectToDatabase())
);

// Supervisor will restart the connection process automatically
```

### External Service Recovery

```java
// Supervised HTTP client process
Proc<HttpClientState, HttpMsg> httpClient = Proc.start(
    initialState -> msg -> {
        return switch (msg) {
            case Get(var url, var reply) -> {
                try {
                    String response = state.client().get(url).body();
                    reply.complete(Result.ok(response));
                    yield state;
                } catch (IOException | InterruptedException e) {
                    // Network error — crash and retry
                    throw new RuntimeException("HTTP request failed", e);
                }
            }
        };
    },
    new HttpClientState(HttpClient.newHttpClient())
);
```

## Production Case Study

### McLaren ATLAS SQL Race

**Problem:** Legacy Java bridge crashed on ECU timeout, dropping entire parameter groups.

**Solution:** JOTP supervision with ONE_FOR_ONE restart.

```java
Supervisor acquisitionSupervisor = Supervisor.builder()
    .name("acquisition-supervisor")
    .strategy(Supervisor.Strategy.ONE_FOR_ONE)
    .maxRestarts(5)
    .timeWindow(Duration.ofSeconds(10))
    .onCrash((child, reason, restartCount) -> {
        telemetry.log("PDA crash: " + child.name() +
                     ", reason: " + reason +
                     ", restart #" + restartCount);
    })
    .build();

// Spawn one PDA per parameter
for (SqlRaceParameter param : parameters) {
    acquisitionSupervisor.supervise(
        param.identifier(),  // "vCar:Chassis"
        () -> ParameterDataAccess.spawn(param, channel)
    );
}
```

**Results:**
- **Crash recovery:** 4,000ms → 50ms (80× faster)
- **Session corruption:** 3-5 per race weekend → 0
- **Manual intervention:** Required → Automatic
- **Code lines:** 12,400 → 3,100 (−75%)

## Best Practices

### 1. Design for Failure

```java
// Good: Idempotent operations survive restart
Proc<PaymentState, PaymentMsg> payment = Proc.start(
    initialState -> msg -> {
        return switch (msg) {
            case Charge(var orderId, var amount) -> {
                if (state.wasCharged(orderId)) {
                    // Already processed — idempotent
                    yield state;
                }
                yield state.charge(orderId, amount);
            }
        };
    },
    new PaymentState()
);
```

### 2. Use Supervision Hierarchies

```java
// Layered supervision for better fault isolation
Supervisor root = Supervisor.builder()
    .name("root-supervisor")
    .strategy(Supervisor.Strategy.ONE_FOR_ONE)
    .build();

Supervisor acquisition = root.superviseChild(
    "acquisition",
    () -> AcquisitionSupervisor.create()
);

Supervisor analytics = root.superviseChild(
    "analytics",
    () -> AnalyticsSupervisor.create()
);
```

### 3. Monitor Restart Counts

```java
supervisor.onCrash((child, reason, restartCount) -> {
    if (restartCount > 10) {
        // Too many restarts — escalate
        alertingService.page("Excessive restarts: " + child.name());
    }
});
```
