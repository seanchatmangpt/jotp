# Distributed State - State Management Across Nodes

import { Diagram } from '@/components/diagram';

## Overview

Distributed state management in JOTP uses **Raft consensus** replicated across JVMs, providing strong consistency for critical data (payment totals, inventory counts, distributed locks) while maintaining the OTP message-passing model.

## The Challenge: State Without Distribution

Local OTP processes maintain state in memory:

```java
// Local process — state lives in one JVM
Proc<InventoryState, InventoryMsg> inventory = Proc.start(
    initialState -> msg -> {
        // State mutation is local
        return switch (msg) {
            case Reserve(var sku, var qty) -> {
                if (state.quantity() >= qty) {
                    yield state.withQuantity(state.quantity() - qty);
                }
                yield state; // Insufficient stock
            }
        };
    },
    new InventoryState("SKU-123", 100)
);
```

**Problem:** If the JVM crashes, the state is lost.

## Solution: Raft-Over-Actors

JOTP replicates state using the **Raft consensus algorithm** — where every Raft RPC is an actor message.

## Raft Message Hierarchy

### Sealed Interface for Raft Messages

```java
public sealed interface RaftMsg<S> permits
    RaftMsg.AppendEntries<S>,
    RaftMsg.AppendEntriesAck,
    RaftMsg.RequestVote,
    RaftMsg.VoteGranted,
    RaftMsg.ClientWrite<S>,
    RaftMsg.ClientRead,
    RaftMsg.ClientRedirect {}

// Leader → Followers: Log replication
record AppendEntries<S>(
    long term,
    NodeId leaderId,
    long prevLogIndex,
    long prevLogTerm,
    List<LogEntry<S>> entries,
    long leaderCommit
) implements RaftMsg<S> {}

// Follower → Leader: Acknowledgment
record AppendEntriesAck(
    long term,
    boolean success,
    long matchIndex
) implements RaftMsg<Object> {}

// Candidate → All: Vote request
record RequestVote(
    long term,
    NodeId candidateId,
    long lastLogIndex,
    long lastLogTerm
) implements RaftMsg<Object> {}

// Follower → Candidate: Vote granted
record VoteGranted(
    long term,
    NodeId voterId
) implements RaftMsg<Object> {}

// Client → Leader: Write request
record ClientWrite<S>(
    S stateDelta,
    CompletableFuture<Result<S, Exception>> reply
) implements RaftMsg<S> {}
```

### State Machine as Pure Function

```java
// Raft state machine: pure function over state and messages
BiFunction<RaftState<S>, RaftMsg<S>, Transition<RaftState<S>>> raftMachine =
    (state, msg) -> {
        return switch (msg) {
            case AppendEntries<S>(var term, var leaderId, var prevLogIndex,
                                  var prevLogTerm, var entries, var leaderCommit) -> {
                // Validate log consistency
                if (term < state.currentTerm()) {
                    yield Transition.reply(new AppendEntriesAck(state.currentTerm(), false, 0));
                }

                // Append entries if consistent
                List<LogEntry<S>> newLog = appendEntries(state.log(), prevLogIndex, prevLogTerm, entries);

                // Update commit index
                long newCommit = Math.min(leaderCommit, newLog.size() - 1);

                yield Transition.nextState(state
                    .withCurrentTerm(term)
                    .withLog(newLog)
                    .withCommitIndex(newCommit)
                );
            }

            case RequestVote(var term, var candidateId, var lastLogIndex, var lastLogTerm) -> {
                // Grant vote if candidate's log is at least as up-to-date
                if (term > state.currentTerm() && isLogUpToDate(state, lastLogIndex, lastLogTerm)) {
                    yield Transition.nextState(state
                        .withCurrentTerm(term)
                        .withVotedFor(candidateId)
                    );
                }
                yield Transition.keepState();
            }

            // ... other cases
        };
    };
```

## Leader Election

### Election Timeout and Randomization

```java
// Election timeout: randomized 150-300ms (Erlang-style)
Thread.ofVirtual().start(() -> {
    Random random = new Random();
    while (running) {
        long timeout = 150 + random.nextInt(150); // 150-300ms

        try {
            Thread.sleep(timeout);
        } catch (InterruptedException e) {
            break;
        }

        // Timeout expired — start election
        startElection();
    }
});
```

## Client Interaction

### Write Path

```java
DistributedStateActor<InventoryState> inventory =
    registry.lookupReplicated("inventory-service");

// Write — routed to Raft leader, replicated to quorum
Result<InventoryState, Exception> result =
    inventory.write(new InventoryMsg.Reserve("SKU-123", 10));

// Internally:
// 1. Client sends ClientWrite to current leader
// 2. Leader appends entry to log
// 3. Leader replicates to followers via AppendEntries
// 4. Leader waits for quorum (2/3 nodes ACK)
// 5. Leader commits entry and applies to state machine
// 6. Leader replies to client
```

### Read Path

```java
// Eventual consistency read (fast, from local follower)
InventoryState snapshot = inventory.read();

// Linearizable read (slower, guarantees latest committed data)
InventoryState latest = inventory.readLinearizable();
```

## Performance & Tradeoffs

### Latency Breakdown

| Operation | Latency (3 nodes) | Latency (5 nodes) |
|-----------|-------------------|-------------------|
| **Write** | ~2-5ms | ~3-7ms |
| - Client → Leader | 0.1ms | 0.1ms |
| - Leader log append | 0.01ms | 0.01ms |
| - Replicate to followers | 1-2ms | 2-4ms |
| - Wait for quorum | 1-2ms | 1-2ms |
| - Commit & reply | 0.1ms | 0.1ms |
| **Read (eventual)** | ~0.1ms | ~0.1ms |
| **Read (linearizable)** | ~2-3ms | ~3-5ms |

### Throughput

| Configuration | Writes/sec | Reads/sec |
|---------------|------------|-----------|
| 3 nodes (1 leader, 2 followers) | 50K/s | 200K/s |
| 5 nodes (1 leader, 4 followers) | 45K/s | 250K/s |
| 7 nodes (1 leader, 6 followers) | 40K/s | 300K/s |

## State Machine Example

### Distributed Inventory

```java
// Domain state
record InventoryState(
    String sku,
    int quantity,
    Map<String, Integer> reservations
) {}

// Raft state machine
DistributedStateActor<InventoryState> inventory =
    DistributedStateActor.create(
        "inventory-service",
        new InventoryState("SKU-123", 100, Map.of()),
        (state, delta) -> {
            return switch (delta) {
                case InventoryMsg.Reserve(var sku, var qty, var orderId) -> {
                    if (state.quantity() >= qty) {
                        Map<String, Integer> newReservations = new HashMap<>(state.reservations());
                        newReservations.put(orderId, qty);
                        yield new InventoryState(
                            sku,
                            state.quantity() - qty,
                            newReservations
                        );
                    }
                    yield state; // Insufficient stock
                }
                case InventoryMsg.Restock(var sku, var qty) -> {
                    yield new InventoryState(sku, state.quantity() + qty, state.reservations());
                }
            };
        },
        clusterConfig
    );
```

## Snapshotting

### Log Compaction

```java
// Snapshot the state machine to prevent unbounded log growth
inventory.snapshot(state -> {
    byte[] serialized = serializer.serialize(state);
    storage.write("inventory-snapshot-" + state.lastAppliedIndex(), serialized);

    // Discard log entries before snapshot
    return new SnapshotMetadata(
        state.lastAppliedIndex(),
        state.lastAppliedTerm(),
        serialized.length
    );
});
```
