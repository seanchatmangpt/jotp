# Message Bus Architecture - Enterprise Messaging Patterns

import { Diagram } from '@/components/diagram';

## Overview

JOTP's MessageBus implements Enterprise Integration Patterns (EIP) with topic-based routing, wildcards, dead letter queues, and durable subscriptions — all built on virtual threads and the OTP supervision model.

## Core Pattern: Message Bus

**Definition:** A MessageBus provides a common communication system that multiple applications use, decoupling publishers from subscribers.

```java
MessageBus bus = MessageBus.create("telemetry-bus");

// Subscribe to telemetry samples
Subscription sub = bus.subscribe("telemetry.samples", envelope -> {
    SampleEvent sample = (SampleEvent) envelope.payload();
    processSample(sample);
});

// Publish samples
bus.publish("telemetry.samples", new SampleEvent(
    "vCar:Chassis", 237.5, Instant.now()
));
```

### Enterprise Integration Patterns

JOTP implements these EIP patterns:

| Pattern | JOTP Class | Use Case |
|---------|-----------|----------|
| **Message Bus** | `MessageBus` | Decouple publishers from subscribers |
| **Publish-Subscribe** | `MessageBus.subscribe()` | Topic-based fan-out |
| **Content-Based Router** | `RouterPatterns` | Route by message content |
| **Scatter-Gather** | `ScatterGatherPatterns` | Parallel requests with aggregation |
| **Correlation Identifier** | `CorrelationPatterns` | Async request-reply matching |
| **Dead Letter Channel** | `DeadLetterHandler` | Failed message handling |
| **Message Store** | `MessageStore` | Durable message persistence |

## Pattern Implementations

### 1. Message Bus with Wildcards

```java
MessageBus bus = MessageBus.create("telemetry-bus");

// Exact topic subscription
bus.subscribe("telemetry.samples.vcar", sample -> {
    System.out.println("vCar: " + sample.value());
});

// Wildcard: telemetry.samples.* (single level)
bus.subscribe("telemetry.samples.*", sample -> {
    // Receives telemetry.samples.vcar, telemetry.samples.nengine, etc.
    log(sample);
});

// Wildcard: telemetry.> (multi-level)
bus.subscribe("telemetry.>", envelope -> {
    // Receives telemetry.samples.*, telemetry.laps.*, etc.
    auditLog.add(envelope);
});
```

### 2. Content-Based Router

```java
RouterPatterns<LogMessage> router = RouterPatterns.create();

router.route(
    msg -> "ERROR".equals(msg.level()),
    msg -> alertingService.sendAlert(msg)
);

router.route(
    msg -> "WARN".equals(msg.level()),
    msg -> metricsService.recordWarning(msg)
);

router.fallback(msg -> log.info(msg.message()));

// Route incoming messages
router.routeMessage(new LogMessage("ERROR", "Connection failed", Instant.now()));
```

### 3. Scatter-Gather

```java
ScatterGatherPatterns<PriceRequest, PriceQuote> scatter =
    ScatterGatherPatterns.<PriceRequest, PriceQuote>create()
        .withTimeout(Duration.ofSeconds(2));

scatter.addHandler(req -> amazonClient.getQuote(req));
scatter.addHandler(req -> walmartClient.getQuote(req));
scatter.addHandler(req -> targetClient.getQuote(req));

// Get quotes from all vendors
var result = scatter.scatterAndGather(new PriceRequest("SKU-123"));

if (result.anySucceeded()) {
    PriceQuote best = result.results().stream()
        .min(Comparator.comparing(PriceQuote::price))
        .orElseThrow();

    System.out.println("Best price: " + best.price());
}
```

### 4. Correlation Identifier

```java
CorrelationPatterns<OrderRequest, OrderConfirmation> correlator =
    CorrelationPatterns.<OrderRequest, OrderConfirmation>create()
        .withDefaultTimeout(Duration.ofSeconds(60));

// Client: Send order and wait
OrderConfirmation confirmation = correlator.sendAndWaitForReply(
    new OrderRequest("ORD-123", List.of("ITEM-A", "ITEM-B"))
);

// Server: Process and reply
void onOrderProcessed(OrderConfirmation confirmation) {
    correlator.handleReply(confirmation.orderId(), confirmation);
}
```

### Gossip Protocol for Message Dissemination

```java
sealed interface GossipMsg permits
    GossipMsg.Publish,
    GossipMsg.Subscribe,
    GossipMsg.Unsubscribe,
    GossipMsg.Sync {}

record Publish(String topic, Envelope envelope) implements GossipMsg {}
record Subscribe(String nodeId, String topic) implements GossipMsg {}
record Unsubscribe(String nodeId, String topic) implements GossipMsg {}
record Sync(Set<String> topics) implements GossipMsg {}

// Gossip handler (runs on each node)
void handleGossip(GossipMsg msg) {
    switch (msg) {
        case GossipMsg.Publish(var topic, var envelope) -> {
            // Deliver to local subscribers
            deliverLocally(topic, envelope);
            // Forward to peers (TTL limited to prevent loops)
            forwardToPeers(msg, envelope.ttl());
        }
        case GossipMsg.Subscribe(var nodeId, var topic) -> {
            // Track remote subscription
            remoteSubscriptions.add(nodeId, topic);
        }
        // ...
    }
}
```

### Backpressure Strategy

```java
// Flow control for slow consumers
MessageBus bus = MessageBus.builder()
    .name("telemetry-bus")
    .backpressureStrategy(BackpressureStrategy.DROP_OLDEST)
    .maxQueueSize(10_000)
    .build();
```

**Strategies:**
- `DROP_OLDEST` — Drop oldest messages when queue full (default)
- `DROP_NEWEST` — Drop incoming messages when queue full
- `BLOCK` — Block publisher until queue has space
- `FAIL` — Throw exception when queue full

## Dead Letter Queue

### Handling Failed Messages

```java
MessageBus bus = MessageBus.builder()
    .name("telemetry-bus")
    .deadLetterHandler(DeadLetterHandler.store(
        MessageStore.file("/var/lib/jotp/dlq")
    ))
    .build();

// Or custom handler
bus = MessageBus.builder()
    .deadLetterHandler((envelope, error) -> {
        if (envelope.payload() instanceof CriticalEvent) {
            // Page on-call
            pagerDuty.triggerPager(envelope, error);
        } else {
            // Log and continue
            log.error("Failed to deliver: " + envelope.topic(), error);
        }
    })
    .build();
```

### DLQ Message Format

```java
Envelope dlqEnvelope = envelope
    .withHeader("dlq.error", error.getMessage())
    .withHeader("dlq.timestamp", Instant.now().toString())
    .withHeader("dlq.original-topic", envelope.topic())
    .withHeader("dlq.node-id", NodeId.current().toString());
```

## Message Persistence

### Durable Subscriptions

```java
// Messages persisted for offline subscribers
MessageStore store = MessageStore.builder()
    .type(MessageStore.Type.POSTGRESQL)
    .connection("jdbc:postgresql://db:5432/jotp")
    .table("messages")
    .retention(Duration.ofDays(7))
    .build();

MessageBus bus = MessageBus.create("telemetry-bus", store);

// Offline subscriber reconnects and receives missed messages
Subscription sub = bus.subscribeDurable(
    "telemetry.samples",
    "subscriber-123",  // Unique subscriber ID
    envelope -> process(envelope),
    Duration.ofHours(24)  // Replay last 24 hours
);
```

## Production Patterns

### Circuit Breaker Integration

```java
MessageBus bus = MessageBus.create("external-api-bus");

CircuitBreaker breaker = CircuitBreaker.create()
    .failureThreshold(5)
    .timeout(Duration.ofSeconds(30))
    .resetTimeout(Duration.ofMinutes(1));

bus.subscribe("api.requests", envelope -> {
    breaker.execute(() -> {
        externalApi.call(envelope.payload());
    });
});
```

### Idempotent Processing

```java
bus.subscribe("orders", envelope -> {
    String idempotencyKey = envelope.correlationId().toString();

    // Check if already processed
    if (idempotencyStore.wasProcessed(idempotencyKey)) {
        log.info("Duplicate message: " + idempotencyKey);
        return;
    }

    // Process message
    processOrder((OrderEvent) envelope.payload());

    // Mark as processed
    idempotencyStore.markProcessed(idempotencyKey);
});
```
