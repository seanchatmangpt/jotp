## Overview

JOTP provides a complete implementation of Vaughn Vernon's reactive messaging patterns, adapted from Scala/Akka to Java 26 with virtual threads and sealed types. These patterns demonstrate how Joe Armstrong's Erlang/OTP primitives translate to modern Java concurrency.

### Pattern Categories

- **[Messaging Channels](./channels/)** (3 patterns) - Fundamental channel types for message flow
- **[Message Construction](./construction/)** (4 patterns) - How to structure messages for transmission
- **[Message Routing](./routing/)** (11 patterns) - How to direct messages to destinations
- **[Message Transformation](./transformation/)** (3 patterns) - Converting message formats and content
- **[Integration Endpoints](./endpoints/)** (4 patterns) - How to connect to external systems
- **[System Management](./management/)** (5 patterns) - Reliability, monitoring, and failure handling
- **[Advanced Patterns](./advanced/)** (4 patterns) - Request-reply and correlation

## Getting Started

### Prerequisites

- Java 26 with preview features enabled (`--enable-preview`)
- Maven 4 (or use included Maven Wrapper: `./mvnw`)
- Understanding of basic JOTP primitives ([Proc](../../explanations/architecture), [Supervisor](../../how-to/supervision-and-fault-tolerance))

### Running Pattern Examples

```bash
# Channels
mvnd exec:java -Dexec.mainClass="io.github.seanchatmangpt.jotp.messagepatterns.channel.PointToPointChannelExample"

# Run all pattern tests
mvnd test -Dtest="*Channel*"      # All channel tests
mvnd test -Dtest="*Router*"       # All routing tests
mvnd test -Dtest="*Translator*"   # All transformation tests

# Full suite with integration tests
mvnd verify
```

### Code Example: Full Pattern Integration

```java
// Using multiple patterns together for Atlas API-style data flow:

// 1. Create event source (Polling Consumer)
var queue = PollingConsumer.create(() -> fetchOrderEvent(), 1000);

// 2. Route by type (Data Type Channel)
var router = DataTypeChannel.create();

// 3. Format for transmission (Format Indicator)
var formatted = FormatIndicator.withFormat(event, "JSON");

// 4. Dedup processing (Idempotent Receiver)
var receiver = IdempotentReceiver.create(1000);

// 5. Send to multiple handlers (Recipient List Router)
var handlers = Arrays.asList(orderHandler, analyticsHandler);
RecipientListRouter.broadcastMessage(handlers, formatted);

// 6. Track execution (Correlation ID)
var correlationId = CorrelationId.generate();
var correlated = CorrelationId.withId(formatted, correlationId);

// 7. Guarantee delivery (Guaranteed Delivery)
var durable = GuaranteedDelivery.create(receiver, new PostgresMessageStore());
durable.send(correlated);
```

## Pattern Selection Guide

### When to Use Each Pattern Category

**Channels** - Choose based on message distribution needs:
- **Point-to-Point**: 1:1 communication, ordered delivery
- **Publish-Subscribe**: 1:N broadcast, decoupled consumers
- **Data Type Channel**: Type-safe routing with sealed interfaces

**Construction** - Structure your messages:
- **Command Message**: Imperative requests with expected replies
- **Document Message**: Transfer complex domain objects
- **Claim Check**: Handle large payloads without network bloat
- **Envelope Wrapper**: Add protocol-level headers and routing

**Routing** - Direct messages intelligently:
- **Content-Based Router**: Smart distribution based on content
- **Scatter-Gather**: Parallel request-reply with aggregation
- **Recipient List**: Fan-out to N destinations
- **Aggregator**: Collect partial results into complete message
- **Process Manager**: Orchestrate multi-step distributed sagas

**Endpoints** - Connect to external systems:
- **Polling Consumer**: Integrate systems without push support
- **Competing Consumers**: Load-balanced message processing
- **Selective Consumer**: Filter messages by criteria

**Management** - Ensure reliability:
- **Idempotent Receiver**: Prevent reprocessing duplicates
- **Dead Letter Channel**: Isolate and handle poison pills
- **Wire Tap**: Monitor message flow without affecting delivery
- **Guaranteed Delivery**: Ensure no message loss on crashes
