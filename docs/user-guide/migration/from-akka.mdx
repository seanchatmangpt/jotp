# Migrating from Akka to JOTP: Complete Pattern Migration Guide

A comprehensive guide for Akka developers migrating to JOTP (Java 26 implementation of OTP primitives) with detailed pattern mappings, code examples, and step-by-step migration strategies.

## Overview

JOTP brings Erlang/OTP's battle-tested concurrency patterns to the JVM using Java 26's virtual threads. While Akka was inspired by Erlang/OTP, JOTP provides a more direct mapping to OTP primitives with Java 26's modern language features.

### Key Architectural Differences

| Aspect | Akka | JOTP |
|--------|------|------|
| **Threading Model** | Custom dispatchers with fork-join pools | Java 26 virtual threads (JVM-managed) |
| **Type Safety** | Typed actors (Akka Typed) or runtime checks | Sealed interfaces with compile-time verification |
| **Configuration** | HOCON configuration files | Pure Java code, no external config |
| **State Management** | Mutable actor state | Immutable state records |
| **Message Handling** | Object-oriented (receive blocks) | Functional (pattern matching) |
| **OTP Parity** | Partial implementation | Full 15-primitive OTP parity |

### Core Philosophical Alignment

Both frameworks share these fundamental principles:
- **Message Passing**: Async communication via immutable messages
- **Supervision Trees**: Hierarchical fault tolerance with automatic restart
- **Let It Crash**: Fail-fast with supervisor recovery
- **Location Transparency**: Processes communicate without knowing physical location

## 1. Actor → Proc Migration

### 1.1 Basic Actor Pattern

**Akka Actor:**
```java
public class CounterActor extends AbstractBehavior<CounterCommand> {
    public static class Increment implements CounterCommand {}
    public static class GetCount implements CounterCommand {
        private final ActorRef<Integer> replyTo;
        public GetCount(ActorRef<Integer> replyTo) { this.replyTo = replyTo; }
    }

    private int count = 0;

    public static Behavior<CounterCommand> create() {
        return Behaviors.setup(CounterActor::new);
    }

    private CounterActor(ActorContext<CounterCommand> context) {
        super(context);
    }

    @Override
    public Receive<CounterCommand> createReceive() {
        return newReceiveBuilder()
            .onMessage(Increment.class, msg -> {
                count++;
                return this;
            })
            .onMessage(GetCount.class, msg -> {
                msg.replyTo().tell(count);
                return this;
            })
            .build();
    }
}

// Usage
var system = ActorSystem.create(CounterActor.create(), "counter");
var counter = system.systemActorOf(CounterActor.create(), "counter");
counter.tell(new Increment());
```

**JOTP Proc:**
```java
// Message protocol (sealed interface)
sealed interface CounterMsg permits Increment, GetCount {}
record Increment() implements CounterMsg {}
record GetCount() implements CounterMsg {}

// State (immutable record)
record CounterState(int count) {}

// Process creation
var counter = Proc.spawn(
    new CounterState(0),
    (state, msg) -> switch (msg) {
        case Increment _ -> new CounterState(state.count() + 1);
        case GetCount _ -> state;  // ask() returns state
    }
);

// Usage
counter.tell(new Increment());
int count = counter.ask(new GetCount()).get(1, TimeUnit.SECONDS).count();
```

**Key Migration Points:**
1. **No ActorSystem**: Processes are created directly without a central coordinator
2. **Functional style**: Handler function `(S, M) → S` instead of class-based actor
3. **Explicit state**: Immutable record vs mutable field
4. **No Behaviors**: Simple state transformation function
5. **Ask returns state**: Direct state access instead of reply-to pattern

### 1.2 ActorRef → ProcRef Mapping

**Akka ActorRef:**
```java
public class OrderProcessor {
    private final ActorRef<OrderCommand> orderActor;

    public OrderProcessor(ActorRef<OrderCommand> orderActor) {
        this.orderActor = orderActor;
    }

    public void processOrder(Order order) {
        orderActor.tell(new ProcessOrder(order));
    }
}
```

**JOTP ProcRef:**
```java
public class OrderProcessor {
    private final ProcRef<OrderState, OrderMsg> orderRef;

    public OrderProcessor(ProcRef<OrderState, OrderMsg> orderRef) {
        this.orderRef = orderRef;
    }

    public void processOrder(Order order) {
        orderRef.tell(new ProcessOrder(order));
    }
}
```

**Critical Difference - ProcRef Stability:**
```java
// Akka: ActorRef can become stale
ActorRef<MyActor> ref = system.actorOf(Props.create(MyActor.class));
// If actor dies and restarts, ref might be stale

// JOTP: ProcRef is stable across restarts
Supervisor sup = Supervisor.create(Strategy.ONE_FOR_ONE, 3, Duration.ofSeconds(60));
ProcRef<MyState, MyMsg> ref = sup.supervise("my-actor", initialState, handler);
// If process crashes and restarts, ref automatically points to new process
ref.tell(new Message()); // Always works, even across restarts
```

### 1.3 Context Handling Migration

**Akka ActorContext:**
```java
public class ContextActor extends AbstractBehavior<Command> {
    private final ActorContext<Command> context;
    private final List<ActorRef<ChildMsg>> children = new ArrayList<>();

    private ContextActor(ActorContext<Command> context) {
        this.context = context;
    }

    @Override
    public Receive<Command> createReceive() {
        return newReceiveBuilder()
            .onMessage(SpawnChild.class, msg -> {
                var child = context.spawn(ChildActor.create(), "child-" + msg.id());
                children.add(child);
                return this;
            })
            .onMessage(GetChildren.class, msg -> {
                msg.replyTo().tell(new ChildrenList(new ArrayList<>(children)));
                return this;
            })
            .onMessage(LogContext.class, msg -> {
                System.out.println("Self: " + context.getSelf());
                System.out.println("Children: " + context.getChildren());
                System.out.println("System: " + context.getSystem());
                return this;
            })
            .build();
    }
}
```

**JOTP State-Based Context:**
```java
// Track children in state instead of context
record ParentState(Map<String, ProcRef<ChildState, ChildMsg>> children) {}

sealed interface Command permits SpawnChild, GetChildren, LogContext {}
record SpawnChild(String id) implements Command {}
record GetChildren(CompletableFuture<Map<String, ProcRef<ChildState, ChildMsg>>> replyTo) implements Command {}
record LogContext() implements Command {}

var parent = Proc.spawn(
    new ParentState(Map.of()),
    (state, msg) -> switch (msg) {
        case SpawnChild(var id) -> {
            var child = Proc.spawn(new ChildState(), childHandler);
            var childRef = new ProcRef<>(child);
            var newChildren = new HashMap<>(state.children());
            newChildren.put(id, childRef);
            yield new ParentState(Map.copyOf(newChildren));
        }
        case GetChildren(var replyTo) -> {
            replyTo.complete(state.children());
            yield state;
        }
        case LogContext _ -> {
            System.out.println("Children tracked in state: " + state.children().keySet());
            yield state;
        }
    }
);
```

## 2. Tell/Bang (!) → tell() Pattern Migration

### 2.1 Fire-and-Forget Messaging

**Akka Tell (!):**
```java
// Akka's ! operator is tell()
actorRef ! new Message("hello");
// or
actorRef.tell(new Message("hello"));

// With dispatcher
actorRef.tell(new Message("hello"), system.dispatcher());

// Fire-and-forget with no guarantee of delivery
actorRef.tell(new Message("hello"));
```

**JOTP tell():**
```java
// Direct tell
proc.tell(new Message("hello"));

// ProcRef tell (stable across restarts)
procRef.tell(new Message("hello"));

// Multiple tells (guaranteed order per sender)
proc.tell(new Message("first"));
proc.tell(new Message("second"));
proc.tell(new Message("third"));
// Process handles in order: first, second, third
```

**Message Ordering Guarantees:**
```java
// Both frameworks guarantee FIFO ordering per sender
// Akka:
actor1.tell(msg1);
actor1.tell(msg2);
// actor1 processes msg1 then msg2

// JOTP:
proc1.tell(msg1);
proc1.tell(msg2);
// proc1 processes msg1 then msg2
```

### 2.2 Pipeline Pattern

**Akka Pipeline:**
```java
public class PipelineActor extends AbstractBehavior<PipelineCommand> {
    @Override
    public Receive<PipelineCommand> createReceive() {
        return newReceiveBuilder()
            .onMessage(ProcessData.class, msg -> {
                // Stage 1
                var stage1 = transformStage1(msg.data());
                // Tell to next stage
                stage2Actor.tell(new ProcessStage2(stage1));
                return this;
            })
            .build();
    }
}
```

**JOTP Pipeline:**
```java
sealed interface PipelineMsg permits ProcessData, ProcessStage2 {}
record ProcessData(String data) implements PipelineMsg {}
record ProcessStage2(String data) implements PipelineMsg {}

var stage1 = Proc.spawn(
    new Stage1State(),
    (state, msg) -> switch (msg) {
        case ProcessData(var data) -> {
            var transformed = transformStage1(data);
            stage2Ref.tell(new ProcessStage2(transformed));
            yield state;
        }
    }
);
```

## 3. Ask (?) → Proc.ask() Pattern Migration

### 3.1 Basic Ask Pattern

**Akka Ask (?):**
```java
public class AskExample {
    private final ActorSystem system;

    public CompletionStage<String> askForData(ActorRef<DataActor> dataActor) {
        return AskWith.ask(
            dataActor,
            replyTo -> new GetData(replyTo),
            Duration.ofSeconds(5),
            system.scheduler()
        );
    }

    public CompletionStage<Integer> askForCount(ActorRef<CounterActor> counter) {
        return AskWith.ask(
            counter,
            replyTo -> new GetCount(replyTo),
            Duration.ofSeconds(3),
            system.scheduler()
        );
    }
}
```

**JOTP ask():**
```java
public class AskExample {
    public CompletableFuture<String> askForData(Proc<DataState, DataMsg> dataProc) {
        return dataProc.ask(new GetData())
            .thenApply(state -> state.currentData());
    }

    public CompletableFuture<Integer> askForCount(Proc<CounterState, CounterMsg> counter) {
        return counter.ask(new GetCount())
            .thenApply(state -> state.count());
    }
}
```

**Key Differences:**
1. **Built-in**: `ask()` is a core method, not a utility
2. **Direct state**: Returns state directly, no reply-to wrapper
3. **No timeout in signature**: Timeout handled by CompletableFuture

### 3.2 Ask with Timeout

**Akka Ask with Timeout:**
```java
public class AskWithTimeout {
    public CompletionStage<Response> askWithTimeout(ActorRef<MyActor> actor) {
        return AskWith.ask(
            actor,
            replyTo -> new Request(replyTo),
            Duration.ofSeconds(10),  // Timeout
            system.scheduler()
        ).handle((response, throwable) -> {
            if (throwable != null) {
                return new Response.Timeout();
            }
            return response;
        });
    }
}
```

**JOTP Ask with Timeout:**
```java
public class AskWithTimeout {
    public CompletableFuture<Response> askWithTimeout(Proc<MyState, MyMsg> proc) {
        return proc.ask(new Request())
            .orTimeout(10, TimeUnit.SECONDS)
            .handle((state, throwable) -> {
                if (throwable != null) {
                    return new Response.Timeout();
                }
                return new Response.Success(state.data());
            });
    }
}
```

### 3.3 Ask Pattern for Request-Reply

**Akka Request-Reply:**
```java
// Message protocol with replyTo
public class UserActor {
    public static class GetUser {
        public final String userId;
        public final ActorRef<User> replyTo;

        public GetUser(String userId, ActorRef<User> replyTo) {
            this.userId = userId;
            this.replyTo = replyTo;
        }
    }

    @Override
    public Receive<Command> createReceive() {
        return newReceiveBuilder()
            .onMessage(GetUser.class, msg -> {
                var user = userRepository.find(msg.userId);
                msg.replyTo().tell(user);
                return this;
            })
            .build();
    }
}

// Client usage
var response = AskWith.ask(
    userActor,
    replyTo -> new GetUser("user123", replyTo),
    Duration.ofSeconds(5),
    system.scheduler()
);
```

**JOTP Request-Reply:**
```java
// Message protocol without replyTo
sealed interface UserMsg permits GetUser {}
record GetUser(String userId) implements UserMsg {}

// Process
var userProc = Proc.spawn(
    new UserState(Map.of()),
    (state, msg) -> switch (msg) {
        case GetUser(var userId) -> {
            var user = state.users().get(userId);
            // ask() returns the entire state, client extracts what they need
            yield state;
        }
    }
);

// Client usage
var state = userProc.ask(new GetUser("user123"))
    .get(5, TimeUnit.SECONDS);
var user = state.users().get("user123");
```

## 4. Supervisor Strategies → JOTP Supervisor

### 4.1 Basic Supervision

**Akka Supervision:**
```java
public class WorkerSupervisor extends AbstractBehavior<Command> {
    @Override
    public Receive<Command> createReceive() {
        return newReceiveBuilder()
            .onMessage(CreateWorker.class, msg -> {
                var worker = context.spawn(
                    Behaviors.supervise(WorkerActor.create())
                        .onFailure[Exception].restart(),
                    "worker-" + msg.id()
                );
                msg.replyTo().tell(worker);
                return this;
            })
            .build();
    }
}
```

**JOTP Supervision:**
```java
var supervisor = Supervisor.create(
    Supervisor.Strategy.ONE_FOR_ONE,
    5,  // maxRestarts
    Duration.ofSeconds(60)  // time window
);

var workerRef = supervisor.supervise(
    "worker-1",
    new WorkerState(),
    workerHandler
);

// Worker is now supervised - automatic restart on crash
```

## 5. Akka Streams → JOTP Channels

### 5.1 Source → Proc Stream

**Akka Source:**
```java
Source<Integer, NotUsed> source = Source.range(1, 100);

source
    .map(x -> x * 2)
    .filter(x -> x > 50)
    .runWith(Sink.foreach(System.out::println), materializer);
```

**JOTP Channel Stream:**
```java
// Create a channel for streaming
var channel = new DataTypeChannel<Integer>(10);

// Producer process
var producer = Proc.spawn(
    new ProducerState(),
    (state, msg) -> switch (msg) {
        case Produce _ -> {
            for (int i = 1; i <= 100; i++) {
                channel.publish(i);
            }
            yield state;
        }
    }
);

// Consumer process
var consumer = Proc.spawn(
    new ConsumerState(),
    (state, msg) -> switch (msg) {
        case Consume _ -> {
            var data = channel.subscribe().receive();
            if (data > 50) {
                System.out.println(data * 2);
            }
            yield state;
        }
    }
);
```

### 5.2 Flow → Proc Pipeline

**Akka Flow:**

```java
Source.range(1, 100)
```

**JOTP Pipeline:**

```java
var stage1 = Proc.spawn(
```

## 7. Persistence → Event Sourcing

### 7.1 Event Sourcing Pattern

**Akka Persistence:**
```java
public class PersistentCounter extends AbstractPersistentBehavior<Command, Event, State> {
    @Override
    public State handleCommand(Command command) {
        return switch (command) {
            case Increment i -> {
                persist(new Event.Incremented(), event -> new State(state.count() + 1));
            }
        };
    }

    @Override
    public State applyEvent(Event event) {
        return switch (event) {
            case Incremented _ -> new State(state.count() + 1);
        };
    }
}
```

**JOTP Event Sourcing:**
```java
// Event store
sealed interface CounterEvent permits Incremented {}
record Incremented(int previousCount, int newCount) implements CounterEvent {}

// Event store (could be database, Kafka, etc.)
var eventStore = new EventStore();

// Process with event sourcing
var counter = Proc.spawn(
    new CounterState(0),
    (state, msg) -> switch (msg) {
        case Increment _ -> {
            var newState = new CounterState(state.count() + 1);
            var event = new Incremented(state.count(), newState.count());
            eventStore.append(event);
            yield newState;
        }
        case GetHistory _ -> {
            var events = eventStore.getEvents();
            // Send history to requester
            yield state;
        }
    }
);

// Recovery
var initialState = eventStore.replayInitialState();
var recoveredCounter = Proc.spawn(initialState, counterHandler);
```

## 8. Testing Approach Migration

### 8.1 Unit Testing

**Akka TestKit:**
```java
class CounterTest extends JUnit5TestKit {
    @Test

        var probe = testKit.createTestProbe();

    @Test

}
```

**JOTP Testing:**
```java
class CounterTest {
    @Test
    void testIncrement() {
        var counter = Proc.spawn(new CounterState(0), counterHandler);
        counter.tell(new Increment());

        var result = counter.ask(new GetCount()).join();
        assertEquals(1, result.count());
    }

    @Test
    void testSupervision() {
        var supervisor = Supervisor.create(
            Supervisor.Strategy.ONE_FOR_ONE,
            5,
            Duration.ofSeconds(60)
        );

        var childRef = supervisor.supervise(
            "child",
            new ChildState(),
            (state, msg) -> switch (msg) {
                case Crash _ -> throw new RuntimeException("Crashed!");
                case Ping _ -> state;
            }
        );

        // Crash the child
        childRef.proc().tell(new Crash());

        // Wait for restart
        Thread.sleep(200);

        // Verify child is still running
        assertTrue(childRef.proc().isRunning());
        assertTrue(childRef.proc().lastError() instanceof RuntimeException);
    }

    @Test

            new SlowState(),

                }
```

### Scenario: Order Processing System

**Original Akka Implementation:**

```java
// Akka Order System
public class OrderSystem {
    private final ActorSystem system;
    private final ActorRef<OrderCommand> orderProcessor;
    private final ActorRef<PaymentCommand> paymentProcessor;
    private final ActorRef<InventoryCommand> inventoryProcessor;

    public OrderSystem() {
        this.system = ActorSystem.create(Behaviors.empty(), "order-system");

        this.inventoryProcessor = system.actorOf(
            Props.create(InventoryActor.class), "inventory"
        );

        this.paymentProcessor = system.actorOf(
            Props.create(PaymentActor.class), "payment"
        );

        this.orderProcessor = system.actorOf(
            Props.create(OrderActor.class,
                this.inventoryProcessor,
                this.paymentProcessor),
            "orders"
        );
    }

    public CompletionStage<OrderConfirmation> processOrder(Order order) {
        return AskWith.ask(
            orderProcessor,
            replyTo -> new ProcessOrder(order, replyTo),
            Duration.ofSeconds(30),
            system.scheduler()
        );
    }
}

public class OrderActor extends AbstractBehavior<OrderCommand> {
    private final ActorRef<InventoryCommand> inventory;
    private final ActorRef<PaymentCommand> payment;

    public OrderActor(ActorRef<InventoryCommand> inventory,
                     ActorRef<PaymentCommand> payment) {
        this.inventory = inventory;
        this.payment = payment;
    }

    @Override
    public Receive<OrderCommand> createReceive() {
        return newReceiveBuilder()
            .onMessage(ProcessOrder.class, msg -> {
                // Step 1: Check inventory
                inventory.tell(new CheckInventory(msg.order(), getContext().getSelf()));
                return this;
            })
            .onMessage(InventoryAvailable.class, msg -> {
                // Step 2: Process payment
                payment.tell(new ProcessPayment(msg.order(), getContext().getSelf()));
                return this;
            })
            .onMessage(PaymentProcessed.class, msg -> {
                // Step 3: Confirm order
                msg.replyTo().tell(new OrderConfirmation(msg.order().getId()));
                return this;
            })
            .build();
    }
}
```

**Migrated JOTP Implementation:**

```java
// JOTP Order System
public class OrderSystem {
    private final ProcRef<InventoryState, InventoryMsg> inventoryRef;
    private final ProcRef<PaymentState, PaymentMsg> paymentRef;
    private final ProcRef<OrderState, OrderMsg> orderRef;

    public OrderSystem() {
        // Create supervisor for fault tolerance
        var supervisor = Supervisor.create(
            Supervisor.Strategy.ONE_FOR_ONE,
            5,
            Duration.ofMinutes(1)
        );

        // Create inventory processor
        var inventoryProc = Proc.spawn(
            new InventoryState(Map.of()),
            inventoryHandler
        );
        this.inventoryRef = supervisor.supervise(
            "inventory",
            () -> new InventoryState(Map.of()),
            inventoryHandler
        );

        // Create payment processor
        this.paymentRef = supervisor.supervise(
            "payment",
            () -> new PaymentState(Map.of()),
            paymentHandler
        );

        // Create order processor
        this.orderRef = supervisor.supervise(
            "orders",
            () -> new OrderState(Map.of(), inventoryRef, paymentRef),
            orderHandler
        );
    }

    public CompletableFuture<OrderConfirmation> processOrder(Order order) {
        return orderRef.ask(new ProcessOrder(order))
            .thenApply(state -> {
                var confirmation = state.orders().get(order.getId());
                if (confirmation != null) {
                    return confirmation;
                }
                throw new RuntimeException("Order processing failed");
            });
    }
}

// Message protocols
sealed interface OrderMsg permits ProcessOrder, InventoryAvailable, PaymentProcessed {}
record ProcessOrder(Order order) implements OrderMsg {}
record InventoryAvailable(Order order, InventoryItem item) implements OrderMsg {}
record PaymentProcessed(Order order) implements OrderMsg {}

sealed interface InventoryMsg permits CheckInventory, InventoryReserved {}
record CheckInventory(Order order) implements InventoryMsg {}
record InventoryReserved(Order order, InventoryItem item) implements InventoryMsg {}

sealed interface PaymentMsg permits ProcessPayment, PaymentConfirmed {}
record ProcessPayment(Order order) implements PaymentMsg {}
record PaymentConfirmed(Order order) implements PaymentMsg {}

// State
record OrderState(
    Map<String, OrderConfirmation> orders,
    ProcRef<InventoryState, InventoryMsg> inventory,
    ProcRef<PaymentState, PaymentMsg> payment
) {}

record InventoryState(Map<String, InventoryItem> items) {}
record PaymentState(Map<String, PaymentTransaction> transactions) {}

// Handlers
BiFunction<InventoryState, InventoryMsg, InventoryState> inventoryHandler =
    (state, msg) -> switch (msg) {
        case CheckInventory(var order) -> {
            var item = state.items().get(order.getItemId());
            if (item != null && item.quantity() >= order.getQuantity()) {
                // Notify order processor
                // (In real implementation, we'd need to track requester)
                yield state;
            }
            yield state;
        }
        case InventoryReserved(var order, var item) -> {
            var newItems = new HashMap<>(state.items());
            newItems.put(item.id(), new InventoryItem(
                item.id(),
                item.quantity() - order.getQuantity()
            ));
            yield new InventoryState(Map.copyOf(newItems));
        }
    };

BiFunction<PaymentState, PaymentMsg, PaymentState> paymentHandler =
    (state, msg) -> switch (msg) {
        case ProcessPayment(var order) -> {
            var transaction = new PaymentTransaction(
                UUID.randomUUID().toString(),
                order.getAmount(),
                "PENDING"
            );
            var newTransactions = new HashMap<>(state.transactions());
            newTransactions.put(transaction.id(), transaction);

            // Process payment (simplified)
            var processedTransaction = new PaymentTransaction(
                transaction.id(),
                transaction.amount(),
                "COMPLETED"
            );
            newTransactions.put(transaction.id(), processedTransaction);

            yield new PaymentState(Map.copyOf(newTransactions));
        }
        case PaymentConfirmed(var order) -> state;
    };

BiFunction<OrderState, OrderMsg, OrderState> orderHandler =
    (state, msg) -> switch (msg) {
        case ProcessOrder(var order) -> {
            // Step 1: Check inventory
            state.inventory().tell(new CheckInventory(order));
            yield state;
        }
        case InventoryAvailable(var order, var item) -> {
            // Step 2: Process payment
            state.payment().tell(new ProcessPayment(order));
            yield state;
        }
        case PaymentProcessed(var order) -> {
            // Step 3: Confirm order
            var confirmation = new OrderConfirmation(
                order.getId(),
                "CONFIRMED",
                Instant.now()
            );
            var newOrders = new HashMap<>(state.orders());
            newOrders.put(order.getId(), confirmation);
            yield new OrderState(Map.copyOf(newOrders), state.inventory(), state.payment());
        }
    };
```

### Pattern 3: FSM → StateMachine

**Akka FSM:**
```java
public class LockFSM extends AbstractFSM<LockState, LockData> {
    {
        startWith(Locked, new LockData(""));

        when(Locked,
            matchEvent(Button.class, (button, data) -> {
                if (data.input().length() >= 4) {
                    return goTo(Open).using(new LockData(""));
                }
                return stay().using(new LockData(data.input() + button.digit()));
            })
        );
    }
}
```

**JOTP StateMachine:**
```java
sealed interface LockState permits Locked, Open {}
record Locked() implements LockState {}
record Open() implements LockState {}

record LockData(String input) {}

var lockMachine = StateMachine.of(
    new Locked(),
    new LockData(""),
    (state, event, data) -> switch (state) {
        case Locked _ -> switch (event) {
            case SMEvent.User(Button(var digit)) -> {
                var newInput = data.input() + digit;
                if (newInput.length() >= 4) {
                    yield Transition.nextState(new Open(), new LockData(""));
                }
                yield Transition.keepState(new LockData(newInput));
            }
            default -> Transition.keepState(data);
        };
        case Open _ -> Transition.nextState(new Locked(), data);
    }
);
```

## 15. Common Pitfalls

### Pitfall 1: Blocking in Handlers
```java
// BAD: Blocks virtual thread
var proc = Proc.spawn(state, (s, m) -> {
    Thread.sleep(1000);  // Don't block!
    return s;
});

// GOOD: Offload blocking work
var executor = Executors.newFixedThreadPool(4);
var proc = Proc.spawn(state, (s, m) -> {
    var future = CompletableFuture.supplyAsync(
        () -> blockingOperation(),
        executor
    );
    return s;
});
```

### Pitfall 2: Mutable State
```java
// BAD: Mutable state in record
record State(List<String> items) {  // List is mutable
    void addItem(String item) {
        items.add(item);  // Don't mutate!
    }
}

// GOOD: Immutable transformations
record State(List<String> items) {
    State addItem(String item) {
        var newItems = new ArrayList<>(this.items);
        newItems.add(item);
        return new State(List.copyOf(newItems));
    }
}
```

### Pitfall 3: Forgetting ProcRef
```java
// BAD: Holding raw Proc reference
Proc<State, Msg> proc = Proc.spawn(state, handler);
// If proc crashes and restarts, this reference is stale

// GOOD: Use ProcRef for supervised processes
var supervisor = Supervisor.create(ONE_FOR_ONE, 5, Duration.ofSeconds(60));
ProcRef<State, Msg> ref = supervisor.supervise("name", state, handler);
// ref.proc() always returns the current process
```

## Conclusion

JOTP provides a simpler, more type-safe alternative to Akka with direct OTP parity. The migration requires shifting from object-oriented to functional style, but the core patterns remain similar. Key advantages include:

1. **Simpler Mental Model**: No ActorSystem, no configuration files, pure Java code
2. **Better Type Safety**: Sealed interfaces with exhaustive pattern matching
3. **Superior Performance**: Virtual threads provide better scalability
4. **Full OTP Parity**: All 15 primitives implemented correctly
5. **Modern Java**: Leverages Java 26 features throughout

The migration path is straightforward: start with simple actors, convert to functional handlers, add supervision, and gradually adopt advanced patterns. The effort is typically 8-12 weeks for a medium-sized codebase.

**Remember**: JOTP is not just "Akka with virtual threads" — it's a direct implementation of Erlang/OTP primitives in Java, with all the battle-tested patterns that have powered massive telecom systems for decades.
