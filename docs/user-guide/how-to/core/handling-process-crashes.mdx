# Handling Process Crashes

> **Problem:** In production systems, processes crash due to network failures, bugs, or resource exhaustion. Without supervision, crashed processes leave holes in your system — requests hang, services become unavailable, and manual intervention is required.

> **Solution:** JOTP's `Supervisor` provides Erlang/OTP-style fault tolerance with automatic crash detection and restart strategies — **"Let It Crash"** philosophy in action.

## When to Use This Pattern

Use supervision when you need:
- **Automatic recovery** from transient failures (network glitches, temporary unavailability)
- **Fault isolation** — one component's crash doesn't cascade to entire system
- **Self-healing systems** that recover without human intervention
- **Stateful service restart** with clean initialization
- **Production-grade resilience** meeting 99.95%+ SLA requirements

## Mental Model

A supervisor is a **parent process** that manages **child processes**:

```
Supervisor (parent)
├─ Child-1 (worker)
├─ Child-2 (worker)
└─ Child-3 (worker)
```

**When a child crashes:**
1. Supervisor detects crash (via virtual thread exception)
2. Applies restart strategy (ONE_FOR_ONE, ONE_FOR_ALL, or REST_FOR_ONE)
3. Restarts child with fresh initial state
4. Tracks restart count to detect crash loops

## Step 1: Choose Restart Strategy

### ONE_FOR_ONE — Independent Workers

**Rule:** Only the crashed child restarts. Siblings unaffected.

**Use case:** Pool of independent workers (HTTP handlers, database connections).

```java
import io.github.seanchatmangpt.jotp.*;
import java.time.Duration;

var supervisor = new Supervisor(
    Supervisor.Strategy.ONE_FOR_ONE,  // Restart only crashed child
    5,                                  // Max 5 restarts per child
    Duration.ofSeconds(60)              // Within 60-second window
);
```

**When to use:**
- Children are **independent** (no shared state)
- One child's crash shouldn't affect others
- Most common strategy (80% of cases)

### ONE_FOR_ALL — Tight Coupling

**Rule:** If ANY child crashes, stop ALL children and restart ALL.

**Use case:** Tightly coupled services where partial state is invalid.

```java
var supervisor = new Supervisor(
    Supervisor.Strategy.ONE_FOR_ALL,   // Restart all on any crash
    3,                                  // Max 3 complete restarts
    Duration.ofMinutes(5)               // Within 5-minute window
);
```

**When to use:**
- Children have **strict dependencies** (database + cache that depends on it)
- Partial restart leads to **inconsistent state**
- Children must start in **specific order**

### REST_FOR_ONE — Ordered Dependencies

**Rule:** Crashed child + all children started AFTER it restart.

**Use case:** Hierarchical dependencies (Config → Connection → Service).

```java
var supervisor = new Supervisor(
    Supervisor.Strategy.REST_FOR_ONE,  // Restart crashed + later children
    5,                                  // Max 5 restarts
    Duration.ofSeconds(30)              // Within 30-second window
);
```

**When to use:**
- Startup order matters (A → B → C)
- Child-C depends on Child-B but not vice versa
- **NOT** fully coupled like ONE_FOR_ALL

## Step 2: Define Child Processes

Children are standard `Proc<S,M>` instances with message handlers:

```java
// Sealed message types
sealed interface WorkerMsg permits Process, GetStatus {}
record Process(String taskId) implements WorkerMsg {}
record GetStatus() implements WorkerMsg {}

// Worker state
record WorkerState(String id, int processed) {}

// Message handler
BiFunction<WorkerState, WorkerMsg, WorkerState> handler =
    (state, msg) -> switch (msg) {
        case Process(var taskId) -> {
            System.out.println(state.id() + " processing: " + taskId);

            // Simulate crash on bad input
            if (taskId.equals("crash")) {
                throw new RuntimeException("Simulated crash!");
            }

            yield new WorkerState(state.id(), state.processed() + 1);
        }
        case GetStatus _ -> state;
    };
```

## Step 3: Register Children with Supervisor

Use `supervisor.supervise()` to register children:

```java
public class WorkerPool {
    private final Supervisor supervisor;
    private final List<ProcRef<WorkerState, WorkerMsg>> workers;

    public WorkerPool(int poolSize) {
        this.supervisor = new Supervisor(
            Supervisor.Strategy.ONE_FOR_ONE,
            5, Duration.ofSeconds(60)
        );
        this.workers = new ArrayList<>();

        // Spawn workers under supervision
        for (int i = 0; i < poolSize; i++) {
            var workerId = "worker-" + i;
            var workerRef = supervisor.supervise(
                workerId,
                new WorkerState(workerId, 0),
                handler
            );
            workers.add(workerRef);
        }
    }

    public ProcRef<WorkerState, WorkerMsg> getNextWorker() {
        // Simple round-robin
        return workers.get(counter++ % workers.size());
    }

    public void shutdown() throws InterruptedException {
        supervisor.shutdown();
    }
}
```

## Step 4: Handle Crashes Gracefully

When a child crashes, supervisor automatically restarts it. Use `ProcRef` which survives restarts:

```java
var pool = new WorkerPool(3);
var worker1 = pool.getNextWorker();

// Worker processes normally
worker1.tell(new Process("task-1"));
Thread.sleep(50);
var status1 = worker1.ask(new GetStatus())
                      .get(1, TimeUnit.SECONDS);
System.out.println("Processed: " + status1.processed());  // 1

// Worker crashes
worker1.tell(new Process("crash"));
Thread.sleep(100);  // Give supervisor time to restart

// ProcRef still valid — points to restarted worker
worker1.tell(new Process("task-2"));
var status2 = worker1.ask(new GetStatus())
                      .get(1, TimeUnit.SECONDS);
System.out.println("After restart: " + status2.processed());  // 1 (fresh state)
```

**Key insight:** `ProcRef` transparently follows restarts. You don't need to update references.

## Complete Working Example: Fault-Tolerant API Gateway

```java
import io.github.seanchatmangpt.jotp.*;
import java.time.Duration;
import java.util.*;
import java.util.concurrent.*;

public class ApiGateway {
    // Messages
    sealed interface GatewayMsg permits HandleRequest, HealthCheck, Shutdown {}
    record HandleRequest(String requestId, String path) implements GatewayMsg {}
    record HealthCheck() implements GatewayMsg {}
    record Shutdown() implements GatewayMsg {}

    // Handler state
    record GatewayState(
        Map<String, Integer> pathCounts,
        List<String> recentErrors
    ) {
        GatewayState recordRequest(String path) {
            var newCounts = new HashMap<>(pathCounts);
            newCounts.merge(path, 1, Integer::sum);
            return new GatewayState(newCounts, recentErrors);
        }
        GatewayState recordError(String error) {
            var newErrors = new ArrayList<>(recentErrors);
            newErrors.add(error);
            if (newErrors.size() > 10) newErrors.remove(0);
            return new GatewayState(pathCounts, newErrors);
        }
    }

    public static void main(String[] args) throws Exception {
        // Create supervisor for API handlers
        var supervisor = new Supervisor(
            Supervisor.Strategy.ONE_FOR_ONE,
            5, Duration.ofSeconds(30)
        );

        // Spawn multiple handler instances
        var handlers = new ArrayList<ProcRef<GatewayState, GatewayMsg>>();
        for (int i = 0; i < 3; i++) {
            var handler = supervisor.supervise(
                "handler-" + i,
                new GatewayState(new HashMap<>(), new ArrayList<>()),
                (state, msg) -> switch (msg) {
                    case HandleRequest(var reqId, var path) -> {
                        System.out.println("[" + Thread.currentThread().getName() +
                                          "] Handling: " + path);

                        // Simulate crash on /error path
                        if (path.equals("/error")) {
                            throw new RuntimeException("Handler crash on /error");
                        }

                        // Simulate processing
                        Thread.sleep(50);
                        yield state.recordRequest(path);
                    }
                    case HealthCheck _ -> {
                        System.out.println("Health check OK");
                        yield state;
                    }
                    case Shutdown _ -> {
                        System.out.println("Handler shutting down");
                        yield state;
                    }
                }
            );
            handlers.add(handler);
        }

        // Simulate requests
        var executor = Executors.newScheduledThreadPool(1);
        var requestCounter = new AtomicInteger(0);

        executor.scheduleAtFixedRate(() -> {
            var handler = handlers.get(requestCounter.get() % handlers.size());
            var reqId = "req-" + requestCounter.incrementAndGet();

            // Mix of normal and error requests
            var path = (requestCounter.get() % 5 == 0) ? "/error" : "/api/users";

            try {
                handler.tell(new HandleRequest(reqId, path));
            } catch (Exception e) {
                System.out.println("Request failed (expected during restart): " + e.getMessage());
            }
        }, 0, 100, TimeUnit.MILLISECONDS);

        // Run for 5 seconds
        Thread.sleep(5000);

        // Check final state of all handlers
        for (int i = 0; i < handlers.size(); i++) {
            var handler = handlers.get(i);
            try {
                var state = handler.ask(new HealthCheck())
                                   .get(1, TimeUnit.SECONDS);
                System.out.println("Handler-" + i + " stats: " + state.pathCounts());
            } catch (Exception e) {
                System.out.println("Handler-" + i + " failed to respond");
            }
        }

        executor.shutdown();
        supervisor.shutdown();
    }
}
```

**Output:**
```
[VirtualThread[x]] Handling: /api/users
[VirtualThread[y]] Handling: /api/users
[VirtualThread[z]] Handling: /error
Handler crash on /error  // Handler-2 crashes
[VirtualThread[x]] Handling: /api/users
[VirtualThread[y]] Handling: /api/users
[VirtualThread[new2]] Handling: /api/users  // Handler-2 restarted, back in rotation
...
Handler-0 stats: {/api/users=17}
Handler-1 stats: {/api/users=16}
Handler-2 stats: {/api/users=14}  // Restarted handler has fewer requests (fresh state)
```

## Pattern: Crash Recovery with State Persistence

By default, restarted children start with **fresh initial state**. For state persistence:

```java
sealed interface DbMsg permits Load, Save, Process {}
record Load() implements DbMsg {}
record Save(Data data) implements DbMsg {}
record Process(String cmd) implements DbMsg {}

var supervisedProc = supervisor.supervise(
    "persistent-worker",
    new WorkerState(null),  // Start with empty state
    (state, msg) -> switch (msg) {
        case Load _ -> {
            // Reload state from database on restart
            var persisted = database.loadState();
            System.out.println("Reloaded state: " + persisted);
            yield new WorkerState(persisted);
        }
        case Process(var cmd) -> {
            var newState = processCommand(cmd, state.data());
            // Persist after each command
            database.saveState(newState);
            yield new WorkerState(newState);
        }
        case Save(var data) -> {
            database.saveState(data);
            yield new WorkerState(data);
        }
    }
);

// On startup: trigger state load
supervisedProc.tell(new Load());
```

## Pattern: Exponential Backoff on Restart

Prevent crash loops by adding backoff in child initialization:

```java
var restartCount = new AtomicInteger(0);

var supervisedProc = supervisor.supervise(
    "worker-with-backoff",
    new WorkerState(),
    (state, msg) -> switch (msg) {
        case Process(var task) -> {
            // ... process task ...
            yield state;
        }
    }
);

// Wrap initialization with backoff
var initProc = Proc.spawn(
    new InitState(),
    (initState, msg) -> {
        var count = restartCount.incrementAndGet();
        var backoffMs = Math.min(1000 * (long)Math.pow(2, count - 1), 30000);

        System.out.println("Starting worker (attempt " + count + "), backoff: " + backoffMs + "ms");
        Thread.sleep(backoffMs);

        // Now ready to receive messages
        supervisedProc.tell(new Process("initial-task"));

        return initState;
    }
);
```

## Testing Crash Recovery

```java
import org.junit.jupiter.api.Test;
import static org.assertj.core.api.Assertions.*;
import static org.awaitility.Awaitility.*;

class SupervisorTest {
    @Test
    void testSupervisorRestartsChildOnCrash() throws Exception {
        var supervisor = new Supervisor(
            Supervisor.Strategy.ONE_FOR_ONE,
            3, Duration.ofSeconds(10)
        );

        var crashCount = new AtomicInteger(0);

        sealed interface TestMsg permits Crash, GetCount {}
        record Crash() implements TestMsg {}
        record GetCount() implements TestMsg {}

        var child = supervisor.supervise(
            "crashy-worker",
            0,
            (state, msg) -> switch (msg) {
                case Crash _ -> {
                    crashCount.incrementAndGet();
                    throw new RuntimeException("Intentional crash");
                }
                case GetCount _ -> state;
            }
        );

        // Verify initial state
        assertThat(child.ask(new GetCount()).get()).isEqualTo(0);

        // Crash the worker
        child.tell(new Crash());
        await().atMost(Duration.ofSeconds(2))
               .until(() -> crashCount.get() == 1);

        // Verify supervisor restarted with fresh state
        assertThat(child.ask(new GetCount()).get()).isEqualTo(0);

        supervisor.shutdown();
    }

    @Test
    void testMaxRestartsExceeded() throws Exception {
        var supervisor = new Supervisor(
            Supervisor.Strategy.ONE_FOR_ONE,
            2, Duration.ofSeconds(5)  // Only 2 restarts allowed
        );

        var child = supervisor.supervise(
            "always-crashes",
            0,
            (state, msg) -> {
                throw new RuntimeException("Always crashes");
            }
        );

        // Trigger crashes
        for (int i = 0; i < 5; i++) {
            child.tell(new TestMsg.Noop());
            Thread.sleep(100);
        }

        // Supervisor should give up and terminate
        await().atMost(Duration.ofSeconds(5))
               .until(() -> !supervisor.isRunning());

        assertThat(supervisor.fatalError()).isNotNull();
    }
}
```

## Common Pitfalls

### 1. Wrong Strategy for Dependencies

**BAD:** ONE_FOR_ONE with shared-state children:
```java
// If ConnectionPool crashes, QueryProcessor still has OLD pool reference
var supervisor = new Supervisor(Strategy.ONE_FOR_ONE, 5, Duration.ofSeconds(60));
var pool = supervisor.supervise("pool", new PoolState(), poolHandler);
var processor = supervisor.supervise("processor",
    new ProcessorState(pool), processorHandler);  // Stale reference!
```

**GOOD:** ONE_FOR_ALL ensures processor gets fresh pool:
```java
var supervisor = new Supervisor(Strategy.ONE_FOR_ALL, 5, Duration.ofSeconds(60));
var pool = supervisor.supervise("pool", new PoolState(), poolHandler);
var processor = supervisor.supervise("processor",
    new ProcessorState(pool), processorHandler);
// If pool crashes, BOTH restart → processor gets fresh pool
```

### 2. Restart Window Too Tight

**BAD:** Too tight — supervisor gives up on transient glitches:
```java
var supervisor = new Supervisor(Strategy.ONE_FOR_ONE, 1, Duration.ofSeconds(1));
// Single network glitch → supervisor terminates
```

**GOOD:** Tolerates transient failures:
```java
var supervisor = new Supervisor(Strategy.ONE_FOR_ONE, 5, Duration.ofSeconds(60));
// Allows 5 restarts within 60 seconds (filters out transient issues)
```

### 3. Not Using ProcRef

**BAD:** Holding direct `Proc` reference:
```java
Proc<State, Msg> proc = new Proc<>(state, handler);  // Unsupervised!
// If proc crashes, reference is stale
```

**GOOD:** Always use `ProcRef` from supervisor:
```java
ProcRef<State, Msg> ref = supervisor.supervise("worker", state, handler);
// ref transparently follows restarts
```

## Monitoring Supervisor Health

```java
// Health check endpoint
public boolean isSupervisorHealthy(Supervisor supervisor) {
    if (!supervisor.isRunning()) {
        log.error("Supervisor down. Last error: {}", supervisor.fatalError());
        alerting.critical("supervisor_down", supervisor.fatalError());
        return false;
    }
    return true;
}

// Prometheus metrics
supervisor.addHealthListener(event -> {
    if (event instanceof SupervisorEvent.ChildRestarted restart) {
        metrics.increment("supervisor.restarts", "child", restart.childId());
    }
    if (event instanceof SupervisorEvent.RestartLimitExceeded) {
        metrics.increment("supervisor.fatal_errors");
    }
});
```
