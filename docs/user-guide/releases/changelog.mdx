## [1.0.0] - 2026-03-23

### Added
**Core OTP Primitives (15 primitives complete)**
- `Proc<S,M>` - Lightweight process with virtual-thread mailbox (gen_server equivalent)
- `Supervisor` - Hierarchical process supervision with ONE_FOR_ONE, ONE_FOR_ALL, REST_FOR_ONE strategies
- `StateMachine<S,E,D>` - Complex workflows with sealed transitions (gen_statem equivalent)
- `EventManager<E>` - Typed event broadcasting (gen_event equivalent)
- `ProcRef<S,M>` - Stable handle that survives supervisor restarts (Pid equivalent)
- `ProcMonitor` - Unilateral DOWN notifications (monitor/2 equivalent)
- `ProcLink` - Bilateral crash propagation (link/2 equivalent)
- `ProcTimer` - Timed message delivery (timer:send_after equivalent)
- `Parallel` - Structured fan-out with fail-fast semantics (pmap/2 equivalent)
- `Result<T,E>` - Railway-oriented error handling ({ok,V}|{error,R} equivalent)
- `CrashRecovery` - Isolated retry with supervised recovery (proc_lib equivalent)
- `ProcRegistry` - Global process name table (global:register_name equivalent)
- `ProcSys` - Live introspection without stopping (sys module equivalent)
- `ProcLib` - Startup handshake init_ack pattern (proc_lib equivalent)
- `ExitSignal` - Exit signal trapping and handling (EXIT signal equivalent)

## Migration Notes

### From 0.9.0 to 1.0.0
**Breaking Changes:**
- `Proc.send()` renamed to `Proc.tell()` (Erlang/OTP naming)
- `Proc.start()` renamed to `Proc.spawn()` (factory pattern)

**Migration:**
```java
// Before (0.9.0)
proc.send(message);
var p = Proc.start(handler, initialState);

// After (1.0.0)
proc.tell(message);
var p = Proc.spawn(handler, initialState);
```

**Benefits:**
- Consistent with Erlang/OTP conventions
- Better factory method patterns
- Improved type safety

### From 0.1.0 to 0.9.0
**Breaking Changes:**
- Requires Java 26 with `--enable-preview`
- Virtual threads mandatory (no traditional thread fallback)

**Migration:**
- Upgrade to Java 26
- Add `--enable-preview` to JVM args
- Remove any thread pool configuration
