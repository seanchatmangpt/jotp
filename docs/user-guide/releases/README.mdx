### Quick Start

```xml
<dependency>
    <groupId>io.github.seanchatmangpt</groupId>
    <artifactId>jotp</artifactId>
    <version>1.0.0</version>
</dependency>
```

```java
import io.github.seanchatmangpt.jotp.*;

var supervisor = new Supervisor(
    Supervisor.Strategy.ONE_FOR_ONE,
    5,
    Duration.ofMinutes(1)
);

ProcRef<Integer, String> counter = supervisor.supervise(
    "counter",
    0,
    (state, msg) -> switch (msg) {
        case "increment" -> state + 1;
        case "reset" -> 0;
        default -> state;
    }
);

counter.tell("increment");
counter.tell("increment");
```

## Release Verification

### Verify Maven Central Release

```bash
# Check if version is available
curl -I https://repo1.maven.org/maven2/io/github/seanchatmangpt/jotp/1.0.0/jotp-1.0.0.pom

# Should return HTTP 200
```

### Verify GPG Signature

```bash
# Download artifact
wget https://repo1.maven.org/maven2/io/github/seanchatmangpt/jotp/1.0.0/jotp-1.0.0.jar

# Download signature
wget https://repo1.maven.org/maven2/io/github/seanchatmangpt/jotp/1.0.0/jotp-1.0.0.jar.asc

# Verify signature
gpg --verify jotp-1.0.0.jar.asc jotp-1.0.0.jar

# Should show "Good signature"
```

### Verify Checksum

```bash
# Download checksums
wget https://repo1.maven.org/maven2/io/github/seanchatmangpt/jotp/1.0.0/jotp-1.0.0.jar.sha512

# Verify
sha512sum -c jotp-1.0.0.jar.sha512

# Should show "OK"
```

## Deprecation Warnings

### Currently Deprecated

**In v1.0.0 (removes in v2.0.0):**

1. **`Proc.start()`** - Use `Proc.spawn()` instead
   ```java
   // Deprecated
   var proc = Proc.start(state, handler);

   // Use this instead
   var proc = Proc.spawn(state, handler);
   ```

2. **`Proc.send()`** - Use `Proc.tell()` instead
   ```java
   // Deprecated
   proc.send(message);

   // Use this instead
   proc.tell(message);
   ```

See [Deprecation Policy](./deprecation-policy.mdx) for complete list.

## Frequently Asked Questions

### General

**Q: How often are releases published?**
A: Major releases annually, minor releases quarterly, patch releases as needed.

**Q: How long is each version supported?**
A: Major versions supported for 2 years after the next major release.

**Q: Are there LTS (Long Term Support) versions?**
A: Not currently, but 1.0.x will receive extended support until 2027.

### Upgrading

**Q: Do I need to change code between patch releases?**
A: No, patch releases (1.0.0 → 1.0.1) are backward compatible.

**Q: What about minor releases?**
A: Minor releases (1.0.0 → 1.1.0) may add features but don't break existing code.

**Q: When do I need to change code?**
A: Only for major releases (1.0.0 → 2.0.0), which have breaking changes. Migration guides are provided.

### Security

**Q: How are security vulnerabilities handled?**
A: See [Security Advisories](./security-advisories.mdx) for the complete process.

**Q: How do I report a security issue?**
A: Email security@seanchatmangpt.io or use GitHub Security Advisory.

**Q: Are old versions still supported?**
A: Only the latest minor version receives security patches.
