# JOTP Archive

This directory contains deprecated or historical documentation preserved for reference purposes.

## Purpose

The archive stores documents that:
- Have been superseded by newer versions
- Contain historical information about the project
- Are kept for backward compatibility reference
- Document deprecated features or approaches

## Archived Content

### Project History
- Migration documentation
- Legacy refactoring notes
- Historical decision records
- Previous architecture versions

### Deprecated Features
- Old API documentation
- Superseded design patterns
- Outdated best practices
- Historical implementation notes

## Usage Guidelines

**DO NOT** use archived documents for:
- Learning current JOTP patterns
- Implementing new features
- Making architectural decisions
- Production code examples

**DO** use archived documents for:
- Understanding project evolution
- Historical research
- Migration from old versions
- Debugging legacy code

## Current Documentation

For current, up-to-date documentation, see:
- **[User Guide](../user-guide/)** - How-to guides and examples
- **[Research](../research/)** - Academic foundations
- **[Roadmap](../roadmap/)** - Implementation plans
- **[Validation](../validation/)** - Test results

## Archival Policy

Documents are moved to archive when:
1. They are >6 months out of date
2. They reference deprecated APIs
3. They have been replaced by comprehensive rewrites
4. They are purely historical in nature

## Retention

Archived documents are retained indefinitely for:
- Historical reference
- Understanding project evolution
- Supporting legacy integrations
- Academic research purposes
