## Archive Philosophy

This archive follows these principles:

1. **Preservation**: Keep historical documentation for reference
2. **Accessibility**: Maintain clear organization and indexing
3. **Context**: Provide archive notices explaining why content was archived
4. **Migration**: Document where current content has moved to
5. **Completion**: Archive completed project documentation

## Accessing Archived Content

### For Historical Reference
- Browse the specific archive directory
- Read the `ARCHIVE_NOTICE.md` for context
- Refer to migration notes for current locations

### For Current Development
- Do NOT use archived content for active development
- Always refer to the main documentation structure
- Check migration notices for updated content

## Archive Maintenance

### Adding New Archives
When archiving new content:
1. Create appropriate subdirectory
2. Move content with archive notice
3. Update this index
4. Cross-reference new locations

### Archive Retention
- **Project Archives**: Keep indefinitely for reference
- **Deprecated Content**: Review annually for removal
- **Temporary Archives**: Remove after migration complete

### Archive Cleanup
- Content archived > 2 years may be considered for removal
- Always check current project relevance before deletion
- Maintain archive of completed work for reference
