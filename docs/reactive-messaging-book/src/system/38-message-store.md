# 38. Message Store

> *"A Message Store persists messages so they can be retrieved later — for audit, replay, aggregation, or claim-check retrieval."*

## Intent

A Message Store provides a persistent (or semi-persistent) registry of messages, keyed by a unique identifier. It is the foundation for several patterns: Claim Check (Pattern 27) stores large payloads here; Message Aggregator (Pattern 20) accumulates partial message groups here; audit logs retain every message for compliance. In JOTP, `MessageStore` wraps a `ConcurrentHashMap<UUID, T>` behind a `Proc` for thread-safe access.

## OTP Analogy

Erlang's ETS (Erlang Term Storage) is the in-process Message Store:

```erlang
%% Create a store
Table = ets:new(message_store, [set, public, named_table]),

%% Store a message
ets:insert(message_store, {MsgId, Payload}),

%% Retrieve
[{MsgId, Payload}] = ets:lookup(message_store, MsgId),

%% Purge
ets:delete(message_store, MsgId).
```

`MessageStore<T>` in JOTP mirrors this API with Java's type system and virtual-thread safety.

## JOTP Implementation

`MessageStore<T>` is backed by a `ConcurrentHashMap<UUID, T>` whose read and write operations are coordinated through a `Proc` mailbox. This ensures that `put`, `get`, and `remove` are linearisable — no message is lost or doubly-returned under concurrent access.

```
Writer ──put(payload)──► Proc<StoreState, StoreCmd>
                              │  ConcurrentHashMap<UUID, T>
Reader ──get(uuid)────────────┤  ← consistent read
Purger ──remove(uuid)─────────┘  ← atomic remove
```

Store lifecycle:
1. **Put**: producer calls `store.put(payload)` → returns `UUID` ticket.
2. **Get**: consumer calls `store.get(uuid)` → returns `T` or `null`.
3. **Remove**: consumer calls `store.remove(uuid)` after processing.
4. **Evict** (TTL): `ProcTimer.sendAfter(ttl, storeProc, Evict(uuid))` purges stale entries.

Key design points:
- The `UUID` key is generated by the store on `put`, not by the caller.
- `get` is non-destructive — the entry remains until explicitly `remove`d.
- Use `purgeOlderThan(Instant)` for bulk TTL eviction.
- For audit use cases, append-only stores never call `remove`; use a `List` or time-series structure instead.

## API Reference

| Method | Description |
|--------|-------------|
| `store.put(payload)` | Store payload; return UUID ticket |
| `store.get(uuid)` | Retrieve payload (null if absent or evicted) |
| `store.remove(uuid)` | Delete entry after consumption |
| `store.size()` | Current number of stored entries |
| `store.keys()` | All current UUIDs (snapshot) |
| `store.purgeOlderThan(instant)` | Evict entries stored before `instant` |

## Code Example

```java
import org.acme.reactive.MessageStore;
import org.acme.Proc;

// --- Message types stored in the store ---
record AuditEvent(
    UUID    eventId,
    String  action,
    String  userId,
    Instant occurredAt,
    Map<String, String> metadata
) {}

// --- Store commands (for Proc-backed implementation) ---
sealed interface StoreCmd<T>
    permits Put, Get, Remove, Purge {}

record Put<T>(T payload, CompletableFuture<UUID> replyTo) implements StoreCmd<T> {}
record Get<T>(UUID id, CompletableFuture<T> replyTo)     implements StoreCmd<T> {}
record Remove<T>(UUID id)                                 implements StoreCmd<T> {}
record Purge<T>(Instant before)                           implements StoreCmd<T> {}

// --- Demo: Audit log using MessageStore ---
public class MessageStoreDemo {

    public static void main(String[] args) throws Exception {
        var store = new MessageStore<AuditEvent>();

        // --- Store lifecycle ---

        // 1. PUT: store messages and receive tickets
        var event1 = new AuditEvent(UUID.randomUUID(), "LOGIN",
            "user-001", Instant.now(), Map.of("ip", "192.168.1.1"));
        var event2 = new AuditEvent(UUID.randomUUID(), "PURCHASE",
            "user-001", Instant.now(), Map.of("orderId", "ORD-001", "amount", "99.99"));
        var event3 = new AuditEvent(UUID.randomUUID(), "LOGOUT",
            "user-001", Instant.now(), Map.of());

        UUID ticket1 = store.put(event1);
        UUID ticket2 = store.put(event2);
        UUID ticket3 = store.put(event3);

        System.out.printf("Stored %d events%n", store.size()); // 3

        // 2. GET: retrieve by ticket (non-destructive)
        AuditEvent retrieved = store.get(ticket2);
        System.out.printf("Retrieved: action=%s userId=%s%n",
            retrieved.action(), retrieved.userId());

        // 3. REMOVE: purge after processing
        store.remove(ticket1);
        System.out.printf("After remove: %d events%n", store.size()); // 2

        // 4. LIST all stored keys
        System.out.println("Remaining keys: " + store.keys());

        // 5. BULK: store all, then batch-process
        var allKeys = store.keys();
        for (UUID key : allKeys) {
            var evt = store.get(key);
            System.out.printf("[AUDIT] %s by %s at %s%n",
                evt.action(), evt.userId(), evt.occurredAt());
            store.remove(key);
        }

        System.out.printf("Store empty: %b%n", store.size() == 0); // true
    }
}
```

## TTL Eviction Pattern

```java
import org.acme.ProcTimer;

// On every put, schedule eviction after 10 minutes
public class TtlMessageStore<T> {
    private final MessageStore<T>  inner = new MessageStore<>();
    private final Duration         ttl;
    private final Proc<?, StoreCmd<T>> storeProc; // the backing Proc

    public TtlMessageStore(Duration ttl, Proc<?, StoreCmd<T>> storeProc) {
        this.ttl       = ttl;
        this.storeProc = storeProc;
    }

    public UUID put(T payload) {
        UUID id = inner.put(payload);
        // Schedule automatic eviction after TTL
        ProcTimer.sendAfter(ttl, storeProc, new Remove<>(id));
        return id;
    }

    public T    get(UUID id)    { return inner.get(id); }
    public void remove(UUID id) { inner.remove(id); }
    public int  size()          { return inner.size(); }
}
```

## Message Aggregation Use Case

```java
// MessageStore as the backing store for MessageAggregator<T>
// Groups are stored until all parts arrive, then assembled

var store = new MessageStore<List<OrderPart>>();

// When a new part arrives for correlationId C:
UUID groupId = getOrCreateGroupId(correlationId);
List<OrderPart> existing = store.get(groupId);
if (existing == null) existing = new ArrayList<>();
existing.add(newPart);
store.remove(groupId);      // remove old group
store.put(groupId, existing); // store updated group

if (existing.size() == expectedParts) {
    // All parts received — assemble and forward
    downstream.send(assemble(existing));
    store.remove(groupId); // purge completed group
}
```

## Test Pattern

```java
import org.junit.jupiter.api.Test;
import org.assertj.core.api.WithAssertions;

class MessageStoreTest implements WithAssertions {

    @Test
    void putReturnsUniqueTicket() {
        var store = new MessageStore<String>();
        var t1    = store.put("payload-1");
        var t2    = store.put("payload-2");
        assertThat(t1).isNotEqualTo(t2);
        assertThat(store.size()).isEqualTo(2);
    }

    @Test
    void getRetrievesStoredPayload() {
        var store   = new MessageStore<String>();
        var payload = "important message";
        var ticket  = store.put(payload);
        assertThat(store.get(ticket)).isEqualTo(payload);
    }

    @Test
    void getIsNonDestructive() {
        var store  = new MessageStore<String>();
        var ticket = store.put("data");
        store.get(ticket);           // first read
        assertThat(store.get(ticket)).isEqualTo("data"); // still there
    }

    @Test
    void removeEvictsEntry() {
        var store  = new MessageStore<String>();
        var ticket = store.put("to-be-removed");
        store.remove(ticket);
        assertThat(store.get(ticket)).isNull();
        assertThat(store.size()).isEqualTo(0);
    }

    @Test
    void unknownTicketReturnsNull() {
        var store = new MessageStore<String>();
        assertThat(store.get(UUID.randomUUID())).isNull();
    }

    @Test
    void concurrentPutsProduceDistinctTickets() throws Exception {
        var store   = new MessageStore<Integer>();
        var tickets = new CopyOnWriteArrayList<UUID>();

        try (var scope = java.util.concurrent.StructuredTaskScope.open()) {
            for (int i = 0; i < 100; i++) {
                int n = i;
                scope.fork(() -> { tickets.add(store.put(n)); return null; });
            }
            scope.join();
        }

        assertThat(new HashSet<>(tickets)).hasSize(100);
        assertThat(store.size()).isEqualTo(100);
    }
}
```

## Caveats & Trade-offs

**Use when:**
- Messages need to be retrieved after the initial delivery (Claim Check, Aggregation, Replay).
- Audit trails require every message to be archived before and after processing.
- A downstream consumer needs asynchronous access to a message produced earlier.

**Avoid when:**
- Messages are consumed immediately and do not need retrieval — a `PointToPointChannel` suffices.
- You need crash-resilient persistence across JVM restarts — `MessageStore` is in-memory; use a database or Redis.
- The store grows unboundedly — always implement TTL eviction or a capacity cap.
